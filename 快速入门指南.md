# 🚀 智能体配置模板 - 快速入门指南

## 📋 5分钟快速开始

### 步骤 1: 选择您的AI平台

#### ModelScope/Trae CN平台用户
```bash
# 复制配置文件
cp 02-服务配置/trae-mcp-config.json ~/.your-platform-config/
```

#### Claude/通用平台用户
```bash
# 复制配置文件
cp 02-服务配置/claude-mcp-config.json ~/.your-platform-config/
```

### 步骤 2: 选择您的开发框架

#### React管理系统
根据项目复杂度选择：
- **简单项目**: `01-框架配置/react-admin/v1-system-prompt.md`
- **中等项目**: `01-框架配置/react-admin/v2-system-prompt.md`
- **复杂项目**: `01-框架配置/react-admin/v3-system-prompt.md`

### 步骤 3: 配置执行规则

```bash
# 通用规则 (必需)
cp 03-执行规则/通用规则/6A规则.md ~/.your-rules/
cp 03-执行规则/通用规则/上下文规则.md ~/.your-rules/

# 个人规则 (可选)
cp 03-执行规则/个人定制/个人规则.md ~/.your-rules/
```

### 步骤 4: 选择您的专家助手

根据您的当前任务选择对应的专家：

| 任务类型 | 推荐专家 | 位置 |
|---------|---------|------|
| 需求分析 | 需求生成专家 | `04-提示词专家/01-需求分析/` |
| 原型设计 | 看板风格专家 | `04-提示词专家/02-原型设计/PC端/` |
| 代码开发 | 网页生成专家 | `04-提示词专家/04-通用专家/` |
| 测试验证 | 功能测试专家 | `04-提示词专家/03-测试验证/` |
| 文档编写 | PPT生成专家 | `04-提示词专家/04-通用专家/` |

## 🔧 配置详解

### MCP服务配置

#### 必需服务 (建议始终启用)
- `sequential-thinking` - 复杂问题分解
- `memory` - 知识记忆系统
- `chrome-devtools` - 浏览器开发工具
- `web-search-prime` - 实时信息检索

#### 开发专用服务
- `serena` - 智能代码助手
- `playwright` - 自动化测试
- `context7` - 技术文档搜索

#### 设计专用服务
- `mcp-server-chart` - 图表生成
- `MCP-ECharts` - 高级图表
- `zai-mcp-server` - 图像分析

### 提示词专家配置

#### 专家启用方式
1. **直接复制** - 将专家配置文件复制到您的AI平台
2. **内容整合** - 将专家内容整合到您现有的提示词中
3. **模块调用** - 根据任务需求动态调用不同专家

#### 专家组合建议
- **开发团队**: 需求专家 + 系统风格专家 + 自动化测试专家
- **产品团队**: PRD专家 + APP设计专家 + PPT专家
- **个人开发者**: 需求专家 + 简约风格专家 + 网页生成专家

~/
├── .ai-config/
│   ├── mcp-services/          # MCP服务配置

## 🆘 常见问题

### Q1: 如何知道哪些MCP服务已正确安装？
A: 检查您的AI平台的MCP服务列表，确保服务状态为"已连接"。

### Q2: 提示词专家如何与现有工作流集成？
A: 可以将专家配置作为模板，根据具体项目需求进行调整和定制。

### Q3: 配置文件冲突如何解决？
A: 建议使用不同的配置目录，或使用配置管理工具进行隔离。

### Q4: 如何获取更多帮助？
A: 查看项目文档中的详细说明，或参与社区讨论。
