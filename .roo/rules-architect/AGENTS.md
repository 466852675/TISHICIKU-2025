# AGENTS.md - Architect Mode

This file provides guidance to agents when working with code in this repository.

## 项目架构规则（非显而易见）

### 系统架构设计原则
- **四层模块化架构**：框架配置、服务配置、执行规则、提示词专家
- **配置驱动设计**：所有功能通过配置文件驱动，避免硬编码
- **平台无关性**：支持多种AI平台（Claude、ModelScope、Trae CN）
- **专家库模式**：专业化的提示词专家配置，按领域分类

### MCP服务架构约束
- **服务发现机制**：通过JSON配置文件动态加载MCP服务
- **平台适配层**：不同平台使用不同的服务配置文件
- **环境变量隔离**：敏感信息通过环境变量管理，不在配置中硬编码
- **服务依赖管理**：某些MCP服务存在启动顺序依赖关系

### React管理系统架构模式
- **组件分层**：页面组件 → 业务组件 → 通用组件
- **状态管理策略**：全局状态用Zustand，页面状态用useState
- **样式架构**：Tailwind CSS + Ant Design，避免样式冲突
- **路由设计**：基于React Router的声明式路由配置

### 配置文件架构设计
```json
{
  "mcpServers": {
    "service-name": {
      "description": "服务描述",
      "enabled": true,
      "server": {
        "command": "启动命令",
        "args": ["参数列表"],
        "type": "stdio|http|sse"
      },
      "tags": ["标签分类"]
    }
  }
}
```

### 专家库架构模式
- **领域分离**：需求分析、原型设计、测试验证、通用任务
- **版本管理**：v1基础版、v2增强版、v3专业版的渐进式设计
- **模板标准化**：统一的专家文档格式和结构
- **质量保证**：每个专家包含完整的质量检查清单

### 6A工作流架构
- **阶段化设计**：Align → Architect → Atomize → Approve → Automate → Assess
- **文档驱动**：每个阶段生成标准化文档
- **质量门控**：阶段间的质量检查和验证机制
- **迭代优化**：基于评估结果的持续改进

### 技术栈架构决策
- **前端技术栈**：React 18.3.1 + TypeScript 5.8.3 + Vite 6.3.5
- **UI框架选择**：Ant Design 5.26.7 + Tailwind CSS 3.4.17
- **状态管理**：Zustand 5.0.3（轻量级、易集成）
- **构建工具**：Vite（快速启动、热重载）

### 扩展性架构设计
- **插件化专家**：新专家可通过标准模板快速添加
- **配置热更新**：MCP服务配置支持动态加载和更新
- **多平台适配**：通过配置文件适配不同AI平台
- **版本兼容性**：向后兼容的配置文件格式设计

### 安全架构考虑
- **配置文件验证**：JSON Schema验证配置文件格式
- **敏感信息保护**：API密钥等通过环境变量管理
- **服务隔离**：MCP服务运行在独立进程中
- **权限控制**：基于配置的服务启用/禁用机制

### 性能架构优化
- **服务懒加载**：按需启动MCP服务，减少启动时间
- **配置缓存**：缓存解析后的配置文件，避免重复解析
- **并发控制**：限制同时运行的MCP服务数量
- **资源监控**：监控服务资源使用情况

### 部署架构模式
- **配置文件部署**：自动复制配置文件到用户目录
- **服务安装检查**：验证MCP服务是否正确安装
- **环境检测**：自动检测操作系统和平台环境
- **故障恢复**：配置错误时的自动回滚机制

### 集成架构设计
- **AI平台集成**：通过标准MCP协议集成多种AI平台
- **开发工具集成**：与VS Code、Cursor等开发工具集成
- **CI/CD集成**：支持自动化构建和部署流程
- **监控集成**：集成日志收集和性能监控

### 数据流架构
```
用户需求 → 专家选择 → 提示词生成 → AI处理 → 结果输出
    ↓           ↓           ↓           ↓           ↓
  需求分析   专家配置    MCP服务    平台API    质量检查
```

### 错误处理架构
- **配置错误处理**：配置文件格式错误时的友好提示
- **服务故障处理**：MCP服务启动失败时的降级策略
- **平台兼容性**：不同平台兼容性问题的自动检测
- **用户指导**：错误发生时的详细解决建议

### 文档架构设计
- **分层文档**：概览文档 + 详细配置文档 + 使用指南
- **示例驱动**：每个配置都包含完整的使用示例
- **故障排除**：常见问题的详细解决方案
- **最佳实践**：基于实际使用经验的最佳实践指南

这些架构规则确保系统的可扩展性、可维护性和可靠性，为长期发展奠定基础。
