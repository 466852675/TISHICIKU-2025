# AGENTS.md - Ask Mode

This file provides guidance to agents when working with code in this repository.

## 项目文档规则（非显而易见）

### 项目架构理解要点
- **四层模块化架构**：01-框架配置、02-服务配置、03-执行规则、04-提示词专家
- **MCP服务配置**：包含Claude/通用平台和ModelScope/Trae CN两套配置
- **专家库分类**：需求分析、原型设计、测试验证、通用任务四大领域
- **6A执行原则**：准确、分析、可执行、适应、负责、敏捷

### 文档导航关键信息
- **专家选择指南**：04-提示词专家/README.md包含完整的专家选择流程
- **React管理规范**：01-框架配置/react-admin/包含三个版本的详细配置
- **MCP服务说明**：02-服务配置/中的配置文件有详细的服务描述
- **执行规则**：03-执行规则/包含6A工作流和上下文规则

### 常见问题解答模式
- **配置问题**：优先检查vite.config.ts和postcss.config.js的语法
- **MCP服务问题**：验证配置文件格式和服务安装状态
- **布局问题**：检查App.tsx的margin设置和页面容器的padding设置
- **中文开发要求**：所有注释和文档必须使用中文

### 技术栈说明重点
- **React管理系统**：严格的技术栈版本要求（React 18.3.1 + TypeScript 5.8.3）
- **构建工具**：Vite 6.3.5，必须使用默认react()配置
- **UI框架**：Ant Design 5.26.7 + Tailwind CSS 3.4.17
- **状态管理**：Zustand 5.0.3

### 项目特定术语解释
- **MCP**：Model Context Protocol，AI服务配置协议
- **6A原则**：项目执行的核心质量保证原则
- **专家库**：标准化的提示词专家配置集合
- **规格驱动工作流**：基于需求规格的完整开发流程

### 使用场景指导
- **企业管理系统开发**：使用react-admin配置 + 系统风格专家
- **移动应用设计**：选择APP端专家（国网风/弥散风/苹果风）
- **需求分析**：使用需求生成专家或PRD生成专家
- **功能测试**：使用功能测试专家进行完整测试

### 配置文件说明
- **claude-mcp-config.json**：Claude/通用平台MCP服务配置
- **trae-mcp-config.json**：ModelScope/Trae CN平台配置
- **v1/v2/v3-system-prompt.md**：React管理系统不同复杂度配置

### 常见误解澄清
- **这不是代码库**：这是AI开发的配置模板库
- **MCP服务不是项目依赖**：是AI平台的外部服务配置
- **专家文档不是代码**：是AI助手的专业提示词配置
- **配置文件不是给项目用的**：是给AI平台配置的

1. **配置问题** → 检查01-框架配置/react-admin/中的配置说明

3. **使用方法问题** → 参考04-提示词专家/README.md的快速开始指南
4. **开发规范问题** → 查看03-执行规则/中的详细规则
