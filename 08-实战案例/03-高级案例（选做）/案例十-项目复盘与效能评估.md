# 案例十：项目复盘与效能评估

> 🔴 **高级案例（选做）** | **预计耗时**：25分钟  
> **使用专家**：周报生成专家 + 提示词生成专家  
> **配套Skill**：evaluation, advanced-evaluation, context-optimization, xlsx  
> **输出成果**：项目复盘报告、效能评估报告、改进建议

---

## 📋 案例概述

### 案例背景

对一个已完成的软件开发项目进行全面的复盘和效能评估，分析项目成功因素、问题根因，并提出改进建议。

### 学习目标

- 掌握evaluation Skill的评估方法
- 学会使用advanced-evaluation进行多维度评估
- 了解context-optimization的上下文优化

---

## 🚀 多IDE操作指南

### Trae用户

#### 步骤1：收集项目数据

准备项目数据：
```
项目基本信息：
- 项目名称：智能客服系统
- 项目周期：2024.01-2024.06（6个月）
- 团队规模：8人（2前端+3后端+1测试+1产品+1UI+1项目经理）
- 预算：200万元

项目成果：
- 完成需求功能点：95个
- 代码行数：50,000行
- 测试用例：300个
- Bug总数：150个（已修复145个）

时间数据：
- 计划工期：180天
- 实际工期：195天
- 延期：15天（8.3%）

质量数据：
- 代码覆盖率：85%
- 严重Bug数：5个
- 线上故障：1次
- 用户满意度：4.2/5.0
```

#### 步骤2：项目复盘

使用周报生成专家：
```
请生成项目复盘报告：

复盘维度：
1. 目标回顾：
   - 原定目标：6个月交付智能客服系统
   - 实际达成：延期15天交付，功能完成100%

2. 过程分析：
   - 需求变更：项目期间发生23次需求变更
   - 技术难点：NLP模型训练耗时超预期
   - 团队协作：前后端联调效率较低

3. 问题根因：
   - 延期原因：第三方接口延迟+需求变更
   - 质量问题：测试覆盖不足导致线上Bug
   - 成本超支：外包NLP服务费用超预期

4. 经验总结：
   - 成功经验：微服务架构提高了系统稳定性
   - 失败教训：需求管理不规范

5. 改进建议：
   - 建立需求变更控制流程
   - 加强第三方依赖管理
   - 完善自动化测试体系
```

#### 步骤3：效能评估

```
@evaluation 请对项目进行效能评估：

评估维度：
1. 进度绩效
   - 计划完成率：100%
   - 延期率：8.3%
   - 里程碑达成率：80%（4/5个）

2. 质量绩效
   - 缺陷密度：3个/千行代码
   - 严重缺陷率：3.3%
   - 代码覆盖率：85%

3. 成本绩效
   - 预算执行率：110%
   - 人力成本：超支10%
   - 外包成本：超支20%

4. 团队效能
   - 人均产出：6,250行代码/人
   - 需求交付率：95/95=100%
   - Bug修复时效：平均2天

评估方法：
- 使用LLM-as-judge模式
- 对比行业基准数据
- 生成雷达图展示各维度得分
```

#### 步骤4：高级评估

```
@advanced-evaluation 请进行多维度深度评估：

评估框架：
1. 技术维度
   - 架构合理性
   - 代码质量
   - 技术债务
   - 可维护性

2. 业务维度
   - 需求满足度
   - 用户体验
   - 业务价值
   - 市场竞争力

3. 管理维度
   - 项目管理
   - 风险管理
   - 沟通协作
   - 资源利用

4. 创新维度
   - 技术创新
   - 流程创新
   - 工具应用
   - 知识沉淀

评估方法：
-  pairwise comparison（成对比较）
-  rubric-based scoring（基于评分标准）
-  避免position bias（位置偏差）
-  生成详细评估报告
```

#### 步骤5：生成评估报表

```
@xlsx 请生成项目效能评估报表：

工作表1：项目概览
- 项目基本信息
- 关键指标汇总
- 整体评分

工作表2：进度分析
- 计划vs实际对比
- 里程碑达成情况
- 延期原因分析

工作表3：质量分析
- Bug趋势图
- 缺陷分布（按严重程度/模块）
- 代码质量指标

工作表4：成本分析
- 预算执行情况
- 成本构成分析
- 超支原因分析

工作表5：团队效能
- 个人产出统计
- 协作效率指标
- 技能成长评估

工作表6：改进建议
- 问题清单
- 根因分析
- 改进措施
- 责任人和时间节点
```

#### 步骤6：上下文优化

```
@context-optimization 请优化项目复盘文档：

优化目标：
1. 压缩文档体积，保留关键信息
2. 结构化存储，便于检索
3. 生成执行摘要

优化策略：
1. 使用结构化摘要技术
2. 提取关键决策点
3. 建立知识索引
4. 生成FAQ

输出：
- 压缩后的复盘文档
- 关键决策清单
- 可复用的经验教训
- 项目知识库索引
```
