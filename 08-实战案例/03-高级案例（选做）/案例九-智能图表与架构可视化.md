# 案例九：智能图表与架构可视化

> 🔴 **高级案例（选做）** | **预计耗时**：20分钟  
> **使用专家**：draw.io图表生成专家  
> **配套Skill**：canvas-design, notebooklm  
> **输出成果**：系统架构图、流程图、时序图（可编辑）

---

## 📋 案例概述

### 案例背景

为一个微服务架构的电商系统设计完整的架构可视化方案，包括系统架构图、服务调用关系、数据流图等。

### 学习目标

- 掌握draw.io图表生成专家的使用
- 学会使用canvas-design设计可视化图表
- 了解notebooklm的研究辅助能力

---

## 🚀 多IDE操作指南

### Trae用户

#### 步骤1：创建图表专家智能体

使用draw.io图表生成专家：
```
请生成电商微服务架构的系统架构图：

系统组件：
1. 网关层：Nginx + Spring Cloud Gateway
2. 服务层：
   - 用户服务（User Service）
   - 商品服务（Product Service）
   - 订单服务（Order Service）
   - 支付服务（Payment Service）
   - 库存服务（Inventory Service）
3. 数据层：
   - MySQL（用户、订单数据）
   - MongoDB（商品数据）
   - Redis（缓存）
   - Elasticsearch（搜索）
4. 中间件：
   - RabbitMQ（消息队列）
   - Zookeeper（服务注册）

要求：
- 使用draw.io格式（XML）
- 分层清晰（接入层、服务层、数据层）
- 标注服务间的调用关系
- 使用标准图标（AWS/Azure图标库）
- 输出可直接导入draw.io编辑
```

#### 步骤2：生成流程图

```
请生成订单处理流程图：

流程步骤：
1. 用户提交订单
2. 校验库存
3. 创建订单（状态：待支付）
4. 调用支付服务
5. 支付成功回调
6. 扣减库存
7. 发送通知
8. 订单完成

异常流程：
- 库存不足：返回错误
- 支付失败：订单取消
- 支付超时：订单自动取消

要求：
- 使用BPMN标准符号
- 区分正常流程和异常流程
- 标注决策节点
- draw.io可编辑格式
```

#### 步骤3：生成时序图

```
请生成下单流程的时序图：

参与角色：
- 用户（User）
- 前端应用（Frontend）
- API网关（Gateway）
- 订单服务（Order Service）
- 库存服务（Inventory Service）
- 支付服务（Payment Service）
- 数据库（Database）

交互流程：
1. 用户 -> 前端：提交订单
2. 前端 -> 网关：POST /api/orders
3. 网关 -> 订单服务：转发请求
4. 订单服务 -> 库存服务：检查库存
5. 库存服务 -> 数据库：查询库存
6. 库存服务 -> 订单服务：返回库存状态
7. 订单服务 -> 数据库：创建订单
8. 订单服务 -> 支付服务：创建支付单
9. 支付服务 -> 订单服务：返回支付链接
10. 订单服务 -> 网关：返回结果
11. 网关 -> 前端：订单创建成功
12. 前端 -> 用户：显示支付页面

要求：
- 使用PlantUML语法
- 标注消息类型（同步/异步）
- 标注响应时间要求
- 可转换为图片
```

#### 步骤4：使用canvas-design美化

```
@canvas-design 请设计架构图封面：

尺寸：1920x1080px

内容：
- 标题：电商微服务架构
- 副标题：高可用分布式系统设计
- 主要技术栈图标（Spring Cloud、MySQL、Redis、RabbitMQ）
- 装饰元素：连接线、节点

风格：
- 科技蓝配色（#3B82F6）
- 深色背景（#0F172A）
- 现代扁平化设计
```

#### 步骤5：使用notebooklm研究

```
@notebooklm 请帮我研究以下主题：

研究主题：微服务架构最佳实践

研究内容：
1. 服务拆分的原则和粒度
2. 服务间通信方式选择（同步/异步）
3. 数据一致性保障（Saga模式、TCC）
4. 服务发现和负载均衡
5. 熔断降级和限流

输出要求：
- 总结关键要点
- 提供实践案例
- 标注信息来源
- 生成研究报告
```

## 图表类型对比

| 图表类型 | 适用场景 | 工具推荐 |
|---------|---------|---------|
| 系统架构图 | 整体架构展示 | draw.io, Lucidchart |
| 流程图 | 业务流程 | draw.io, Visio |
| 时序图 | 交互流程 | PlantUML, draw.io |
| 类图 | 面向对象设计 | PlantUML, StarUML |
| 状态图 | 状态流转 | draw.io |
| 部署图 | 物理部署 | draw.io |
