# 案例二：电力巡检管理系统需求文档

> 🟢 **基础案例（必做）** | **预计耗时**：10分钟  
> **使用专家**：需求生成专家 + PRD生成专家  
> **配套Skill**：docx, doc-coauthoring  
> **输出成果**：标准的需求A单和PRD文档（Word格式）

---

## 📋 案例概述

### 案例背景

某电力公司需要开发一套电力巡检管理系统，用于管理巡检任务、记录巡检结果、生成巡检报告。需要将模糊的业务需求转化为标准的需求文档。

### 学习目标

- 掌握需求生成专家的使用方法
- 学会编写标准化的PRD文档
- 了解docx Skill的文档生成能力

---

## 🚀 多IDE操作指南

### Trae用户

#### 步骤1：创建智能体

1. 创建智能体，命名为"需求分析专家"
2. 复制 `06-提示词库/01-需求分析/需求生成专家.md` 内容

#### 步骤2：输入原始需求

```
我们需要一个电力巡检管理系统，主要功能包括：

业务背景：
- 公司有50名巡检人员，负责1000个电力设施的巡检
- 目前使用纸质记录，效率低，容易丢失
- 需要实时上传巡检结果，方便管理

功能需求：
1. 任务分配：管理员可以分配巡检任务给巡检员
2. 巡检执行：巡检员手机端接收任务，现场拍照记录
3. 异常上报：发现异常时立即上报，附带照片和位置
4. 报告生成：自动生成巡检报告，支持导出PDF
5. 统计分析：查看巡检完成率、异常统计等

技术要求：
- 移动端APP + Web管理后台
- 支持离线巡检，有网络后自动同步
- GPS定位记录巡检轨迹

请生成标准的需求A单和PRD文档。
```

#### 步骤3：获取文档

- AI会生成结构化的需求文档
- 复制内容保存为Markdown文件

#### 步骤4：使用Skill生成Word

```
@docx 请将刚才生成的PRD文档转换为Word格式，要求：
1. 使用标准国网文档模板
2. 包含目录和页眉页脚
3. 表格样式规范
4. 输出文件：电力巡检系统PRD.docx
```

---

### Cursor用户

1. 在 `.cursorrules` 中添加需求专家规则
2. 或使用Chat直接发送专家提示词+需求
3. 使用Python脚本生成Word：

```bash
python 07-Skill库/skills/docx/scripts/document.py --input PRD.md --output PRD.docx
```
