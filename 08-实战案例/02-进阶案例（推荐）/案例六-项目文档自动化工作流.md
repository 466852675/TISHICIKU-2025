# 案例六：项目文档自动化工作流

> 🟡 **进阶案例（推荐）** | **预计耗时**：30分钟  
> **使用专家**：PPT生成专家 + 周报生成专家 + 提示词生成专家  
> **配套Skill**：docx, pptx, pdf, xlsx, internal-comms  
> **输出成果**：完整的项目文档套件（Word/PPT/PDF/Excel）

---

## 📋 案例概述

### 案例背景

为一个软件开发项目建立完整的文档自动化工作流，包括需求文档、设计文档、周报、汇报PPT等。

### 学习目标

- 掌握4个文档处理Skill的使用
- 学会建立文档自动化工作流
- 了解internal-comms的内部沟通写作

---

## 🚀 多IDE操作指南

### Trae用户

#### 步骤1：创建文档工作流

创建3个智能体：
1. **PPT生成专家**：复制 `06-提示词库/04-通用专家/PPT生成专家.md`
2. **周报生成专家**：复制 `06-提示词库/04-通用专家/周报生成专家.md`
3. **提示词生成专家**：复制 `06-提示词库/04-通用专家/提示词生成专家.md`

#### 步骤2：生成项目周报

使用周报生成专家：
```
请生成本周项目周报：

项目信息：
- 项目名称：智能客服系统开发
- 项目周期：2024.01-2024.06
- 当前阶段：开发阶段（第8周）

本周工作：
1. 完成用户管理模块开发（100%）
2. 完成知识库管理模块（80%）
3. 修复15个Bug
4. 编写单元测试用例50个

下周计划：
1. 完成知识库模块剩余功能
2. 开始对话引擎开发
3. 进行集成测试

风险和问题：
- 第三方NLP接口延迟交付，可能影响进度3天
```

调用Skill生成Word：
```
@docx 请将周报转换为Word格式，要求：
1. 使用公司标准模板
2. 添加项目Logo
3. 表格展示进度数据
4. 输出：项目周报-第8周.docx
```

#### 步骤3：生成项目汇报PPT

使用PPT生成专家：
```
请生成项目阶段性汇报PPT：

汇报主题：智能客服系统开发项目阶段汇报

内容要点：
1. 项目概述和目标
2. 当前进度（甘特图）
3. 已完成功能演示
4. 技术架构图
5. 质量指标（代码覆盖率、Bug统计）
6. 风险和对策
7. 下阶段计划
```

调用Skill：
```
@pptx 请生成项目汇报PPT，要求：
1. 15-20页
2. 包含图表和架构图
3. 使用公司配色
4. 输出：项目阶段汇报.pptx
```

#### 步骤4：生成需求文档PDF

```
@pdf 请将以下需求文档转换为PDF格式：
- 输入：需求规格说明书.md
- 要求：
  1. 添加目录和页码
  2. 使用标准字体
  3. 保留Markdown格式
  4. 添加页眉（公司名称）和页脚（页码）
- 输出：需求规格说明书.pdf
```

#### 步骤5：生成项目数据报表

```
@xlsx 请生成项目数据报表，包含以下工作表：

工作表1：项目进度跟踪
- 任务名称、负责人、计划开始、计划结束、实际进度、状态

工作表2：Bug统计
- 按严重程度分类（致命/严重/一般/提示）
- 按模块分类
- 趋势图（每周新增/解决数量）

工作表3：代码质量
- 代码行数统计
- 代码覆盖率
- 技术债务指标

工作表4：资源投入
- 人力投入（人天）
- 成本统计
- 预算执行率
```

#### 步骤6：内部沟通

```
@internal-comms 请撰写项目延期通知邮件：

背景：
- 由于第三方接口延迟，项目需要延期1周
- 原计划6月30日交付，现调整为7月7日

收件人：项目相关方（产品经理、技术总监、客户代表）

要求：
1. 说明延期原因
2. 展示应对措施
3. 确认新的里程碑
4. 语气专业但诚恳
```
