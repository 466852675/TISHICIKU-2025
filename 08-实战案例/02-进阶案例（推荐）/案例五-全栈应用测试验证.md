# 案例五：全栈应用测试验证

> 🟡 **进阶案例（推荐）** | **预计耗时**：25分钟  
> **使用专家**：功能测试专家 + 自动化测试专家 + 文档质量校验专家  
> **配套Skill**：test-driven-development, systematic-debugging, webapp-testing  
> **输出成果**：测试计划、测试用例、自动化测试脚本、测试报告

---

## 📋 案例概述

### 案例背景

为一个电商管理系统进行全面的测试验证，包括功能测试、自动化测试和文档质量检查。

### 学习目标

- 掌握3个测试专家的使用方法
- 学会编写测试用例和自动化脚本
- 了解TDD和调试Skill的应用

---

## 🚀 多IDE操作指南

### Trae用户

#### 步骤1：创建测试专家智能体

创建3个智能体：
1. **功能测试专家**：复制 `06-提示词库/03-测试验证/功能测试专家.md`
2. **自动化测试专家**：复制 `06-提示词库/03-测试验证/自动化测试专家.md`
3. **文档质量校验专家**：复制 `06-提示词库/03-测试验证/文档质量校验专家.md`

#### 步骤2：功能测试

使用功能测试专家：
```
请为以下电商管理系统生成功能测试用例：

系统模块：
1. 用户管理：注册、登录、权限管理
2. 商品管理：商品上架、编辑、下架
3. 订单管理：下单、支付、发货、退款
4. 库存管理：入库、出库、库存预警
5. 数据统计：销售报表、用户分析

要求：
- 每个模块至少10个测试用例
- 包含正向和反向测试
- 使用标准测试用例格式
- 标注P0/P1/P2优先级
```

#### 步骤3：自动化测试

使用自动化测试专家编写Playwright脚本：
```
请为电商系统的核心流程编写自动化测试：

测试场景：
1. 用户注册登录流程
2. 商品搜索和下单流程
3. 订单支付和查询流程
4. 管理员后台操作

要求：
- 使用Playwright + TypeScript
- 包含Page Object模式
- 添加数据驱动测试
- 生成Allure测试报告
```

调用Skill：
```
@test-driven-development 请为订单模块实施TDD：
1. 编写订单创建功能的单元测试
2. 实现订单创建功能
3. 重构代码并确保测试通过
4. 使用Jest测试框架
```

#### 步骤4：调试和Bug修复

```
@systematic-debugging 请帮我排查以下问题：

问题描述：
用户反馈订单支付后状态未更新，还是显示"待支付"

已知信息：
- 支付回调接口返回成功
- 数据库中有支付记录
- 订单状态字段未更新
- 日志显示回调已处理

请系统性地分析可能的原因并提供解决方案。
```

#### 步骤5：运行自动化测试

```
@webapp-testing 请运行以下测试：
1. 执行Playwright自动化测试脚本
2. 生成测试报告和截图
3. 分析失败的测试用例
4. 提供修复建议
```

#### 步骤6：文档质量检查

使用文档质量校验专家检查测试文档质量。

---

## ✅ 检查清单

- [ ] 功能测试用例覆盖所有模块
- [ ] 自动化测试脚本可运行
- [ ] 测试报告生成成功

## 🎯 进阶练习

1. **性能测试**：使用 `webapp-testing` 进行压力测试
2. **安全测试**：检查SQL注入、XSS等安全漏洞
3. **集成测试**：测试第三方支付接口集成
