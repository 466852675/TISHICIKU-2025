### 2. 业务设计执行脚本

**文件**：[01-AI编程IDE辅助业务设计具体执行脚本.md](file:///c:/Users/hangy/Desktop/AI智能体配置模板套件/01-完整指南/01-AI编程IDE辅助业务设计具体执行脚本.md)

**6A工作流**：

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  A1: Align  │ → │ A2: Architect│ → │ A3: Atomize │
│   对齐需求   │    │   架构设计   │    │   任务分解  │
└─────────────┘    └─────────────┘    └─────────────┘
                                              ↓
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ A6: Assess  │ ← │ A5: Automate │ ← │ A4: Approve │
│   评估复盘   │    │   自动执行   │    │   审核确认  │
└─────────────┘    └─────────────┘    └─────────────┘
```

**各阶段说明**：

| 阶段 | 名称 | 核心任务 | 产出物 | 使用专家 |
|------|------|---------|--------|---------|
| A1 | Align（对齐） | 理解需求、明确目标 | 需求确认单 | 需求生成专家 |
| A2 | Architect（架构） | 技术选型、架构设计 | 架构设计文档 | PRD生成专家 |
| A3 | Atomize（分解） | 任务拆分、排期规划 | 任务清单 | - |
| A4 | Approve（审核） | 方案评审、风险识别 | 评审报告 | 文档质量校验专家 |
| A5 | Automate（执行） | 编码实现、自动化测试 | 可交付代码 | 各开发专家 |
| A6 | Assess（评估） | 质量评估、复盘总结 | 评估报告 | 周报生成专家 |

**配套Skill**：
- planning-with-files：文件化规划
- executing-plans：计划执行
- verification-before-completion：完成前验证
- evaluation：项目评估

## 各IDE使用本模块的方法

### Trae用户

**学习方式**：
1. 在Trae中打开 `00-AI编程IDE零基础小白操作指南.md`
2. 使用AI对话功能，边阅读边提问
3. 跟随第2章的Trae专项教程操作

**实践方式**：
1. 创建"6A工作流助手"智能体
2. 复制 `01-AI编程IDE辅助业务设计具体执行脚本.md` 中的提示词
3. 在实际项目中应用6A工作流

### Cursor用户

**学习方式**：
1. 在Cursor中打开指南文件
2. 使用Composer功能辅助理解
3. 跟随第3章的Cursor专项教程操作

**实践方式**：
1. 在项目根目录创建 `.cursorrules` 文件
2. 添加6A工作流规则
3. 在AI对话中引用规则

### 其他IDE用户

**学习方式**：
1. 使用任何Markdown阅读器打开指南
2. 跟随对应IDE的章节学习
3. 参考第6章的对比分析选择IDE

**实践方式**：
1. 将6A工作流保存为项目文档
2. 在AI对话中引用文档
3. 按步骤执行工作流
