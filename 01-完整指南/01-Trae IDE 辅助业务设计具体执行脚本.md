## 分阶段详细说明

### 阶段一：原始需求收集梳理

#### 📋 阶段信息

| 项目                 | 内容                                                                                       |
| -------------------- | ------------------------------------------------------------------------------------------ |
| **使用智能体** | 无（人工主导）                                                                             |
| **前置依赖**   | 项目立项审批通过、客户对接人确认、调研提纲（含"业务场景/功能需求/非功能需求"模块）准备完成 |
| **输入物**     | 客户业务痛点说明、现场调研纪要、地市专责需求记录（含 Excel 需求清单）                      |
| **产出物**     | 《初步需求说明》                                                                           |
| **过程记录**   | 调研会议录像、客户签字确认的需求记录单                                                     |

#### 🔧 操作说明

1. **组建调研团队**：组建"产品+业务"调研团队
2. **现场调研**：赴客户现场（如国网省营服稽查中心）开展业务访谈或需求调研
3. **形成文档**：整理调研内容，形成《初步需求说明》

#### ✅ 验收标准

- [ ] 调研记录完整
- [ ] 客户签字确认
- [ ] 初步需求说明文档已生成

### 阶段二：原始需求生成需求A单

#### 📋 阶段信息

| 项目                 | 内容                                                      |
| -------------------- | --------------------------------------------------------- |
| **使用智能体** | 需求生成专家                                              |
| **智能体配置** | `06-提示词模板库/01-需求分析/需求生成专家.md`           |
| **前置依赖**   | 阶段一产出物经客户确认签字、Trae 工作区已上传原始需求文件 |
| **输入物**     | 《初步需求说明》《需求功能清单》                          |
| **产出物**     | 《需求A单》                                               |

#### 🔧 操作说明

**步骤1：创建智能体**

1. 在Trae IDE中创建智能体，命名为"需求生成专家"
2. 打开配置文件：`06-提示词模板库/01-需求分析/需求生成专家.md`
3. 复制完整提示词内容到智能体配置中
4. 保存智能体

**步骤2：生成需求A单**

1. 在Trae右侧对话区选择"需求生成专家"
2. 引用输入物文件（使用 `@文件名` 引用）
3. 输入以下需求描述：

```
请根据 [初步需求说明] 内容，遵循国网《需求A单编写规范》，生成包含"项目概述、功能需求（按模块拆分）、非功能需求（响应时间 ≤2s）、验收标准"的需求A单，以 Markdown 格式输出并保存至当前文件夹。
```

### 阶段三：需求A单校验完善

#### 📋 阶段信息

| 项目                 | 内容                                                |
| -------------------- | --------------------------------------------------- |
| **使用智能体** | 文档质量校验专家（+人工辅助）                       |
| **智能体配置** | `06-提示词模板库/03-测试验证/文档质量校验专家.md` |
| **前置依赖**   | 阶段二《需求A单》已生成                             |
| **输入物**     | 《初步需求说明》《需求功能清单》《需求A单》         |
| **产出物**     | 《需求A单校验分析报告》《需求A单（完善版）》        |

#### 🔧 操作说明

**步骤1：创建智能体**

1. 在Trae IDE中创建智能体，命名为"文档质量校验专家"
2. 打开配置文件：`06-提示词模板库/03-测试验证/文档质量校验专家.md`
3. 复制完整提示词内容到智能体配置中
4. 保存智能体

**步骤2：执行校验**

1. 调用"文档质量校验专家"
2. 引用输入物文件（使用 `@文件名` 引用）
3. 输入以下校验需求：

```
请检查 [需求A单] 是否完整覆盖 [初步需求说明] 内容，重点校验：
① 需求编号是否唯一
② 术语是否符合《电力营销系统术语规范》
③ 验收标准是否可量化

生成校验报告，标注"严重/一般/轻微"问题等级。
```

**步骤3：完善文档**

1. 智能体自动生成校验报告
2. 人工针对"严重问题"（如漏项"工单超时预警"功能）补充需求描述
3. 智能体根据人工意见自动优化文档

### 阶段四：需求A单内部评审

#### 📋 阶段信息

| 项目                 | 内容                                             |
| -------------------- | ------------------------------------------------ |
| **使用智能体** | 无（人工审核）                                   |
| **前置依赖**   | 阶段三《需求A单（完善版）》已完成                |
| **输入物**     | 《需求A单（完善版）》、国网标准需求A单 Word 模板 |
| **产出物**     | 《需求A单内部评审意见表》《需求A单（Word版）》   |

#### 🔧 操作说明

**步骤1：格式转换**

将 Markdown 格式需求A单转换为 Word 版（匹配国网模板）

**步骤2：组织评审**

组织"项目经理+产品负责人+技术架构师+业务咨询"评审，重点审核：

- 需求可行性
- 技术实现难度

**步骤3：评审决策**

- **不通过**：如"实时工单统计"需求超出技术能力，返回阶段三修改
- **通过**：进入下一阶段

#### 📋 阶段信息

| 项目                 | 内容                                                        |
| -------------------- | ----------------------------------------------------------- |

#### 🔧 操作说明

**步骤1：准备评审材料**

1. 对原型页面进行脱敏处理（隐藏客户敏感数据字段）
2. 打包A单、原型、PPT形成评审材料

**步骤2：客户汇报**

向客户"项目负责人+业务科室主任+信息部领导"汇报，演示原型功能

**步骤3：评审决策**

- **需求A单不通过**：返回阶段三修改
- **原型不通过**：返回阶段八修改
- **两者均通过**：签署确认单，启动开发
