## 案例1：电费缴纳APP界面设计

### 项目背景
**需求**：设计一个符合国家电网标准的电费缴纳APP界面，包含核心缴费功能。

**目标**：生成可直接在浏览器中运行的HTML原型代码。

### 操作流程

#### 第1步：准备工作（1分钟）
1. 确保已创建"国网APP设计专家"智能体
2. 确认MCP服务配置完成
3. 在项目中创建文件夹：`case1-electricity-app/`

#### 第2步：发起需求（2分钟）
在AI对话框中选择"国网APP设计专家"，输入：

```
我需要设计一个电费缴纳APP，包含以下功能：

1. 首页
   - 本月电费账单（金额、截止日期）
   - 快捷缴费按钮
   - 历史欠费提醒
   - 常用功能入口

2. 账单查询页
   - 本月详细账单
   - 过去12个月账单列表
   - 账单详情查看

3. 在线缴费页
   - 缴费金额输入
   - 支付方式选择（微信、支付宝、银行卡）
   - 支付确认

4. 个人中心
   - 用户信息
   - 缴费记录
   - 消息通知
   - 设置

要求：
- 符合国家电网标准色彩（主色#009D85）
- 适配iPhone 15 Pro屏幕
- 界面美观，交互友好
- 生成完整的HTML代码，可直接运行
- 代码要有详细注释
```

#### 第3步：获取代码（3分钟）
AI会生成完整的HTML代码，包含：
- 页面结构（4个页面）
- CSS样式（Tailwind CSS）
- JavaScript交互逻辑
- 国网标准色彩系统

#### 第4步：保存和运行（2分钟）
1. 在项目中创建文件：`electricity-app.html`
2. 将AI生成的代码粘贴到文件中
3. 保存文件（Ctrl+S）
4. 右键文件 → "在浏览器中打开"

#### 第5步：测试和优化（5分钟）
**测试清单**：
- [ ] 首页账单显示正常
- [ ] 页面切换流畅
- [ ] 缴费流程完整
- [ ] 色彩符合国网标准
- [ ] 移动端适配正常

**优化调整**：
如果需要修改，可以继续对话：
```
请把缴费按钮的颜色改成更深的绿色，并增加一个加载动画效果。
```

#### 第6步：交付成果
**最终文件**：
- `electricity-app.html` - 完整的APP原型
- `case1-electricity-app/README.md` - 项目说明

**演示效果**：
- 可以在手机浏览器中直接运行
- 支持触摸滑动操作
- 完整的缴费流程体验

## 案例2：电力巡检管理系统需求文档

### 项目背景
**需求**：为电力巡检管理系统生成标准的需求A单和PRD文档。

**目标**：产出符合国网标准的完整需求文档，可直接用于评审和开发。

### 操作流程

#### 第1步：准备工作（1分钟）
1. 创建"需求分析专家"智能体
2. 创建项目文件夹：`case2-inspection-system/`
3. 创建文档目录：`case2-inspection-system/docs/`

#### 第2步：需求收集（3分钟）
在AI对话框中选择"需求分析专家"，输入：

```
我们需要开发一个电力巡检管理系统，背景如下：

业务背景：
- 电力设备需要定期巡检，确保安全运行
- 目前使用纸质记录，效率低，易出错
- 巡检结果需要统计分析，支持决策

主要功能需求：
1. 巡检任务管理
   - 任务创建和分配
   - 巡检路线规划
   - 任务进度跟踪

2. 现场巡检执行
   - 移动端数据录入
   - 支持拍照上传
   - 离线数据缓存
   - 异常情况标记

3. 巡检结果统计
   - 完成率统计
   - 问题分类统计
   - 趋势分析
   - 报表导出

4. 异常处理流程
   - 异常报警
   - 处理流程跟踪
   - 整改验证

非功能需求：
- 支持1000+用户并发
- 移动端离线使用
- 数据安全加密
- 响应时间<2秒

请生成完整的需求A单和PRD文档。
```

#### 第3步：生成需求文档（5分钟）
AI会生成两个文档：

**文档1：需求A单**
- 项目概述
- 功能需求清单（详细）
- 非功能需求
- 验收标准
- 技术约束

**文档2：PRD文档**
- 产品功能架构图
- 业务流程图
- 页面流程图
- 数据字典
- API接口定义
- 详细验收标准

#### 第4步：保存文档（2分钟）
1. 创建文件：`docs/需求A单_电力巡检系统.md`
2. 创建文件：`docs/PRD_电力巡检系统.md`
3. 将AI生成的内容分别保存

#### 第5步：质量检查（3分钟）
使用"文档质量校验专家"检查文档：

```
请检查以下需求文档的质量：
1. 完整性：是否覆盖所有需求点
2. 准确性：业务逻辑是否正确
3. 规范性：格式是否符合标准
4. 可执行性：开发团队能否直接使用

[粘贴生成的文档内容]
```

#### 第6步：评审准备（1分钟）
创建评审材料：
- `docs/评审要点.md` - 关键决策点
- `docs/疑问清单.md` - 需要确认的问题

#### 第7步：交付成果
**完整文档集**：
- `需求A单_电力巡检系统.md`
- `PRD_电力巡检系统.md`
- `评审要点.md`
- `疑问清单.md`

**文档特点**：
- 符合国网标准格式
- 内容完整无遗漏
- 逻辑清晰可执行
- 可直接用于团队评审

## 案例3：销售数据可视化看板

### 项目背景
**需求**：设计一个销售数据可视化看板，支持多维度数据分析。

**目标**：生成包含多种图表的响应式HTML看板，支持实时数据筛选。

### 操作流程

#### 第1步：准备工作（1分钟）
1. 创建"数据看板专家"智能体
2. 确认MCP服务包含图表生成工具
3. 创建项目文件夹：`case3-sales-dashboard/`

#### 第2步：需求分析（2分钟）
在AI对话框中选择"数据看板专家"，输入：

```
我需要一个销售数据可视化看板，数据背景如下：

数据指标：
- 销售额（月度、季度、年度）
- 订单数量
- 客户数量
- 转化率
- 平均客单价

维度分析：
- 时间维度：年、季度、月、周
- 区域维度：华北、华东、华南、西部
- 产品维度：A类、B类、C类产品
- 客户维度：新客户、老客户

图表需求：
1. 主仪表盘
   - 关键指标卡片（4个）
   - 销售额趋势图（折线图）
   - 区域占比图（饼图）

2. 详细分析
   - 产品销售对比（柱状图）
   - 转化率趋势（折线图）
   - 客户增长（面积图）
   - 销售漏斗（漏斗图）

3. 筛选功能
   - 时间范围选择
   - 区域筛选
   - 产品类别筛选
   - 客户类型筛选

技术要求：
- 使用ECharts图表库
- 暗色主题设计
- 响应式布局
- 支持数据实时更新
- 交互友好

请生成完整的HTML代码。
```

#### 第3步：生成看板代码（5分钟）
AI会生成包含以下内容的HTML文件：
- 页面布局结构
- ECharts图表配置
- 筛选功能实现
- 暗色主题样式
- 响应式设计代码

#### 第4步：数据准备（3分钟）
AI会提供示例数据格式，你需要准备真实数据：

```javascript
// 示例数据结构
const salesData = {
  // 时间序列数据
  monthly: [
    { month: '2024-01', sales: 1250000, orders: 1250 },
    // ... 更多数据
  ],
  
  // 区域数据
  regional: [
    { region: '华北', sales: 3500000, proportion: 35 },
    // ... 更多数据
  ],
  
  // 产品数据
  products: [
    { category: 'A类', sales: 4500000, growth: 15.2 },
    // ... 更多数据
  ]
}
```

#### 第5步：集成和测试（5分钟）
1. 将代码保存为 `sales-dashboard.html`
2. 替换示例数据为真实数据
3. 在浏览器中测试：
   - [ ] 图表正确显示
   - [ ] 筛选功能正常
   - [ ] 响应式适配正常
   - [ ] 暗色主题美观

#### 第6步：优化调整（3分钟）
根据测试结果，可以要求AI优化：
```
图表显示正常，但希望：
1. 增加数据刷新按钮
2. 优化移动端的图表大小
3. 增加数据导出功能
```

#### 第7步：交付成果
**最终文件**：
- `sales-dashboard.html` - 完整的看板页面
- `data-format.md` - 数据格式说明
- `deployment-guide.md` - 部署指南

**功能特点**：
- 6种图表类型
- 多维度筛选
- 暗色主题
- 响应式设计
- 交互友好

## 案例通用技巧

### 1. 需求描述技巧
**好的需求描述**：
```
设计一个电费缴纳APP，包含账单查询、在线缴费、历史记录功能。
要求：国网标准风格，iPhone 15 Pro适配，可直接运行。
```

**避免的描述**：
```
做个APP界面（太模糊）
```

### 2. 优化调整技巧
**分步骤优化**：
```
第一步：先实现基本功能
第二步：优化视觉设计
第三步：增加交互效果
第四步：性能优化
```

**具体化要求**：
```
❌ "界面要好看"
✅ "使用#009D85主色，增加圆角和阴影，字体使用16px"
```

### 3. 质量检查清单
每个案例完成后，检查：
- [ ] 功能完整性
- [ ] 视觉美观度
- [ ] 代码规范性
- [ ] 文档完整性
- [ ] 可运行验证

### 4. 团队协作建议
- **代码审查**：让同事检查生成的代码
- **用户测试**：让目标用户试用原型
- **文档评审**：需求文档需要团队确认
- **知识沉淀**：将优质代码保存为模板
