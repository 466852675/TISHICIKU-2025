## 环境安装与界面熟悉

### 1.1 下载安装Trae IDE

**步骤1：访问官网或使用安装包**

- 打开浏览器，访问：https://www.trae.cn/
- 或查看项目中的安装指南：`00-工具安装包/安装网址.md`
- 点击"立即下载"按钮

**步骤2：选择版本**

- **Windows用户**：下载 `.exe` 安装包
- **Mac用户**：下载 `.dmg` 安装包

**步骤3：安装过程**

- 双击安装包，按照提示完成安装
- 首次启动时，可以选择"跳过教程"直接进入主界面

**步骤4：验证安装**

- 打开Trae IDE，看到主界面即表示安装成功
- 界面包含：左侧文件浏览器、中间代码编辑区、右侧AI对话区

### 1.2 界面布局说明

```
┌─────────────────────────────────────────────────────────┐
│  文件菜单  项目名称                    设置图标  AI助手  │
├─────────────┬──────────────────────────┬───────────────┤
│             │                          │               │
│  文件浏览器 │      代码编辑区          │   AI对话框    │
│  (左侧)     │      (中间)              │   (右侧)      │
│             │                          │               │
│  - 项目文件 │      代码内容            │   智能体选择  │
│  - 文件夹   │      编辑区域            │   对话历史    │
│             │                          │               │
└─────────────┴──────────────────────────┴───────────────┘
```

**关键区域说明：**

- **右上角设置图标** ⚙️：规则、MCP、智能体配置入口
- **右侧AI对话框**：与AI助手交互的主要区域
- **左下角智能体选择**：切换不同的AI助手
- **左侧文件浏览器**：管理项目文件和文件夹

## 个人规则配置

### 2.1 什么是个人规则？

个人规则是**全局配置**，对所有项目生效，一次设置永久使用。就像给AI助手设定"工作习惯"。

### 2.2 配置步骤

**步骤1：打开规则设置**

1. 在AI对话框右上角，点击 **⚙️ 设置图标**
2. 在弹出菜单中选择 **"规则"**
3. 界面会显示"个人规则"和"项目规则"两个区域

**步骤2：创建个人规则（使用项目模板库中的完整规则）**

1. 在"个人规则"区域，点击 **"+ 创建 user_rules.md"** 按钮
2. 系统会自动创建并打开 `user_rules.md` 文件
3. **打开项目中的个人规则模板文件**：`04-规则配置/通用配置/个人规则.md`
4. **复制完整内容**到 `user_rules.md` 文件中

> 💡 **提示**：项目模板库中的个人规则已经过优化，包含完整的配置说明，建议直接使用。

**步骤3：保存并生效**

- 按 `Ctrl+S` 保存文件
- 规则立即生效，无需重启Trae

### 2.3 验证规则是否生效

在AI对话框中输入：

```
请介绍一下你自己，并说明你遵循什么规则
```

如果AI回复中提到了"使用中文"、"添加注释"等内容，说明规则配置成功。

## 项目创建与项目规则

### 3.1 创建新项目

**步骤1：新建项目**

1. 点击菜单栏 **"文件" → "新建项目"**
2. 输入项目名称：`AI业务设计项目`
3. 选择保存位置：建议创建专门的工作目录
4. 点击"创建"

**步骤2：熟悉项目结构**
创建后，左侧文件浏览器会显示：

```
AI业务设计项目/
├── .trae/              # Trae配置目录（自动创建）
├── src/                # 源代码目录（可选）
├── docs/               # 文档目录（可选）
└── README.md           # 项目说明
```

### 3.2 配置项目规则

**步骤1：打开项目规则设置**

1. 确保已打开项目（左侧文件浏览器显示项目名称）
2. 在AI对话框右上角，点击 **⚙️ 设置图标**
3. 选择 **"规则"**
4. 在"项目规则"区域，点击 **"+ 创建 project_rules.md"**

**步骤2：配置项目规则（使用项目模板库中的完整6A执行规则）**

系统会自动在项目根目录创建 `.trae/rules/project_rules.md` 文件：

1. **打开项目中的6A执行规则模板文件**：`04-规则配置/项目规则/6A执行规则.md`
2. **复制完整内容**到 `.trae/rules/project_rules.md` 文件中

> 💡 **提示**：6A执行规则是项目的核心工作流程，包含完整的6个阶段（对齐→架构→原子化→审批→自动化→评估），建议直接使用项目模板库中的完整版本。

**步骤3：保存并验证**

- 保存文件后，在AI对话框中输入：`@Builder 请帮我检查当前项目的规则配置`
- AI应该能正确识别并应用项目规则

## MCP服务配置

### 4.1 什么是MCP？

MCP（Model Context Protocol）就像给AI助手安装"插件"，让它能够：

- 🧠 **深度思考**：处理复杂逻辑
- 💾 **记忆存储**：记住对话历史
- 🔍 **文档搜索**：查找技术资料
- 🤖 **浏览器操作**：自动测试网页
- 📊 **图表生成**：数据可视化

### 4.2 使用MCP市场（推荐方式）

**步骤1：进入MCP设置**

1. 在AI对话框右上角，点击 **⚙️ 设置图标**
2. 选择 **"MCP"** 选项卡
3. 看到MCP市场和已安装的服务列表

**步骤2：添加MCP服务**

1. 点击 **"添加"** 按钮
2. 在MCP市场中浏览可用服务
3. 推荐添加以下核心服务：

| 服务名称            | 功能描述     | 重要性     | 配置方式 |
| ------------------- | ------------ | ---------- | -------- |
| sequential-thinking | 顺序思考工具 | ⭐⭐⭐⭐⭐ | 一键添加 |
| memory              | 知识图谱记忆 | ⭐⭐⭐⭐⭐ | 一键添加 |
| context7            | 技术文档搜索 | ⭐⭐⭐⭐   | 一键添加 |
| playwright          | 浏览器自动化 | ⭐⭐⭐⭐   | 一键添加 |
| fetch               | HTTP请求工具 | ⭐⭐⭐     | 一键添加 |

**步骤3：配置服务**

- 点击服务右侧的 **"+"** 按钮
- 部分服务需要填写API密钥（按提示操作）
- 点击"确认"完成添加

**步骤4：验证配置**

- 配置完成后，在AI对话框输入：`@Builder with MCP 请列出可用的工具`
- 如果配置成功，AI会回复可用的MCP工具列表

### 4.3 手动配置MCP（高级方式）

如果MCP市场中没有需要的服务，可以手动配置：

**步骤1：进入手动配置**

1. 在MCP设置页面，点击 **"手动添加"**
2. 会看到一个JSON编辑框

**步骤2：使用项目模板库中的MCP配置**

1. **打开项目中的MCP配置文件**：`05-服务配置/MCP服务/通用mcp配置.json`
2. **复制完整配置内容**到编辑框中

> 💡 **提示**：项目模板库中的MCP配置包含7个核心服务（Sequential Thinking、context7、Chrome DevTools、playwright、图表服务等），已经过测试验证，建议直接使用。

**步骤3：保存并验证**

- 点击"保存"按钮
- 重启Trae IDE（完全关闭后重新打开）
- 在AI对话框中测试服务是否正常

### 4.4 常见问题解决

**问题1：MCP服务无法启动**

```bash
# 检查Node.js版本（需要>=14）
node --version

# 如果未安装Node.js，访问 https://nodejs.org 下载安装
```

**问题2：服务添加失败**

- 检查网络连接
- 尝试重新启动Trae IDE
- 确认命令行工具已正确安装

**问题3：配置不生效**

- 检查JSON格式是否正确（使用在线JSON验证工具）
- 确认服务的"enabled"状态为true
- 重启Trae IDE

## 智能体创建与使用

### 5.1 什么是智能体？

智能体（Agent）就像公司里的**专业专家**：

- **需求专家**：帮你写需求文档
- **设计专家**：帮你做界面设计
- **测试专家**：帮你检查质量
- **通用专家**：处理PPT、周报等日常任务

### 5.2 使用内置智能体

**步骤1：选择智能体**

1. 在AI对话框左下角，点击智能体下拉菜单
2. 选择需要的智能体：

| 智能体名称       | 适用场景 | 主要功能             |
| ---------------- | -------- | -------------------- |
| Builder          | 基础编程 | 代码编写、调试、重构 |
| Builder with MCP | 复杂开发 | 调用工具、自动化测试 |
| 对话助手         | 通用问答 | 解答问题、提供建议   |

**步骤2：开始对话**

- 在对话框中输入需求
- AI会根据所选智能体的能力进行回复
- 可以随时切换智能体

### 5.3 创建自定义智能体（专家）

**步骤1：进入创建界面**

1. 在智能体下拉菜单中，点击 **"创建智能体"**
2. 会打开智能体配置页面

**步骤2：填写基本信息**

- **智能体名称**：如"国网APP设计专家"
- **描述**：专注于国家电网标准风格的移动端界面设计

**步骤5：保存并测试**

- 点击"保存"按钮
- 在AI对话框中选择新创建的智能体
- 输入测试需求，验证效果

### 5.4 智能体管理技巧

**修改智能体**

- 在智能体下拉菜单中，点击"管理智能体"
- 选择要修改的智能体，更新提示词

**删除智能体**

- 在管理界面，点击智能体右侧的删除图标
- 确认删除（不可恢复）

**导出/导入**

- 可以将配置好的智能体导出为JSON文件
- 在其他电脑上导入，快速复用

### 案例1：电费缴纳APP界面设计（15分钟）

**目标**：使用"国网APP设计专家"创建一个电费缴纳APP界面

**步骤1：准备环境**

- 确认MCP服务配置完成（参考 `05-服务配置/MCP服务/通用mcp配置.json`）

**步骤2：开始设计**

1. 在AI对话框中选择"国网APP设计专家"
2. 输入需求：
   ```

   1. 账单查询 - 显示本月电费、历史欠费
   ```

**步骤5：优化调整**

- 如果有不满意的地方，可以继续对话优化
- 例如："把主色调改成更深的绿色"、"增加加载动画"

## 常见问题速查

### Q1: 安装Trae后无法启动？

**解决方案：**

1. 检查系统要求：Windows 10+ 或 macOS 10.15+
2. 重新下载安装包，确保下载完整
3. 以管理员身份运行（Windows）
4. 检查防火墙是否阻止了Trae

### Q2: AI对话没有反应或很慢？

**解决方案：**

1. 检查网络连接
2. 确认已选择正确的智能体
3. 重启Trae IDE
4. 检查是否选择了"Builder with MCP"（需要MCP服务）

### Q3: MCP服务无法添加？

**解决方案：**

```bash
# 检查Node.js是否安装
node --version

# 如果未安装，访问 https://nodejs.org 下载LTS版本

# 验证npm是否可用
npm --version
```

### Q4: 规则配置后不生效？

**解决方案：**

1. 检查规则文件是否保存（Ctrl+S）
2. 确认文件路径正确：
   - 个人规则：系统自动创建在用户目录
   - 项目规则：`.trae/rules/project_rules.md`
3. 重新打开AI对话框
4. 输入测试指令验证规则是否生效

### Q5: 智能体创建失败？

**解决方案：**

1. 检查智能体名称是否重复
2. 确保提示词内容不为空
3. 验证MCP工具配置正确（如果关联了工具）
4. 重启Trae后重试

### Q6: 生成的代码无法运行？

**解决方案：**

1. 检查浏览器控制台错误信息
2. 确认依赖库是否正确加载
3. 验证HTML结构是否完整
4. 在AI对话框中反馈问题，要求修复

### Q7: 如何删除或修改已创建的智能体？

**操作步骤：**

1. 在智能体下拉菜单中，点击"管理智能体"
2. 找到要修改的智能体
3. 点击编辑图标修改提示词
4. 或点击删除图标移除智能体

### Q8: 如何备份和迁移配置？

**备份方法：**

1. 个人规则：复制 `user_rules.md` 内容
2. 项目规则：复制 `.trae/rules/project_rules.md`
3. 智能体：导出配置JSON文件
4. MCP配置：复制配置JSON

**迁移方法：**
在新电脑上按相同步骤重新配置，或导入备份文件
