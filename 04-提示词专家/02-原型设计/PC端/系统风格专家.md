## 国网PC端页面设计规范

### 1. 总体设计原则

#### 1.1 四大基本原则

（1）**一致性原则**：公司各子信息系统界面应保持色彩、字体、布局、间距等风格统一，融合为视觉一致、交互一致的公司统一信息系统。

（2）**合理性原则**：信息系统布局应遵循用户从上而下，自左向右浏览的操作习惯，页面模块化划分，常用业务功能模块邻近排列。页面设计应横平竖直、结构清晰、层次分明，页面元素之间符合间距要求、不应过于拥挤。

（3）**简洁性原则**：信息系统界面应避免过度装饰和冗余设计，保持界面简洁明了，菜单深度不宜超过3层。页面展示必要的信息和功能，减少用户的认知负荷。

（4）**灵活性原则**：信息系统界面应能够适应不同的用户需求和使用环境，满足不同用户个性化需求，实现千人千面的用户体验。

#### 1.2 设计主题

业务管理类系统只有PC端，应采用国网绿和科技蓝2套界面设计，支持主题切换功能。

### 2. 技术规范

#### 2.1 屏幕适配

- PC端系统界面分辨率应适配显示屏比例 16:9
- 适配尺寸应兼容 1920x1080像素及以上显示屏

#### 2.2 色彩规范

采用国网绿、科技蓝主题样式，支持主题切换功能。色彩系统包括四大类：主色、辅助色、功能色和中性色。

##### 2.2.1 主色规范

主色应用于图标、主要按钮，顶部导航背景、超链接文字、高亮提醒等。

| 颜色名称 | 色值    | 应用场景                     |
| -------- | ------- | ---------------------------- |
| 国网绿   | #007D3B | 主要用于国网绿主题的核心元素 |
| 科技蓝   | #1D7AD9 | 主要用于科技蓝主题的核心元素 |

##### 2.2.2 辅助色规范

辅助色应用于次要按钮、标签、图表、提示信息等。

| 颜色名称 | 色值    | 应用场景                         |
| -------- | ------- | -------------------------------- |
| 橙色     | #F77E22 | 用于提醒、警告相关的辅助元素     |
| 蓝色     | #4F9DFA | 用于信息、提示相关的辅助元素     |
| 青色     | #A6E2FA | 用于特殊状态或高亮显示的辅助元素 |
| 紫色     | #A67F9E | 用于分类、标识的辅助元素         |

##### 2.2.3 功能色规范

功能色应用于向用户明确的传达成功、警告、错误等信息和状态。

| 颜色名称 | 色值    | 应用场景                     |
| -------- | ------- | ---------------------------- |
| 成功     | #1DAD73 | 用于表示操作成功的状态       |
| 提醒     | #FF754C | 用于表示需要注意的提醒状态   |
| 错误     | #F23A3A | 用于表示操作错误或异常状态   |
| 禁止     | #E53935 | 用于表示禁止、不可操作的状态 |

##### 2.2.4 中性色规范

中性色应用于文字、背景、边框、分割线等。

| 颜色名称 | 色值    | 应用场景                 |
| -------- | ------- | ------------------------ |
| 深色文字 | #333333 | 用于标题、重要文本内容   |
| 灰色文字 | #666666 | 用于正文、普通文本内容   |
| 浅灰文字 | #999999 | 用于辅助说明、提示文本   |
| 更深背景 | #F2F2F2 | 用于区块背景、分割区域   |
| 浅灰背景 | #F8F8F8 | 用于页面背景、容器背景   |
| 白色背景 | #FFFFFF | 用于卡片、弹窗等元素背景 |
| 选中背景 | #E6F1EB | 用于选中状态的背景色     |

##### 2.2.5 色彩应用原则

1. **一致性原则**：同一系统中，相同功能的元素应使用相同的颜色
2. **层级分明**：通过颜色的深浅、明暗来区分信息的层级和重要性
3. **对比度合理**：确保文本与背景之间的对比度符合可读性要求
4. **主题切换**：支持国网绿和科技蓝两套主题的无缝切换

#### 2.3 字体规范

- **中文字体**：微软雅黑
- **英文字体**：Arial
- **数字字体**：DINPro
- **特殊字体**：以图片形式输出
- **行高设置**：字号的1.5倍

**字号规范**：

- 系统名称：24px，粗体（Bold）
- 一级菜单：20px，粗体（Bold）
- 模块标题：16px，粗体（Bold）
- 表格内容、输入框文字等正文内容：14px，常规（Regular）
- 辅助文字、占位文字：14px，常规（Regular）

#### 2.4 图标按钮规范

- 按钮风格保持一致
- 主要操作采用面性风格
- 次要操作采用线性风格
- 圆角弧度为5px

### 3. 页面布局规范

#### 3.1 系统首页布局

系统首页分为三个主要区域：

1. **顶部导航区**

   - 包含系统名称、菜单导航、消息通知、登录账号
   - 左右间距留白32px
   - 菜单导航为下拉方式，每一层菜单包含"默认状态"、"悬浮状态"
   - 登录账号为隐藏面板，划入账号信息显示"系统通知"、"待办事项"、"修改密码"、"退出"
   - 系统通知采用统一图标，红点和数字分别表示"新消息"和消息数量
   - 退出按钮位于顶部导航右侧-划入登录账号展开隐藏面板
2. **菜单路径导航区**

   - 与当前位置和上级路径通过颜色加以区分，突显当前位置
   - 上级路径的链接可点击，当前位置不可点击
3. **内容展示区**

   - 分为上下结构，即顶部固定尺寸区域及内容展示适配区域缩放
   - 由导航区域、内容区域组成
   - 内容区域与其他区域之间间距为16px进行留白
   - 页面框架中内容区域根据实际业务场景可出现纵向滚动条

#### 3.2 典型页面布局

##### 3.2.1 页面总览

- 全景总览页面分为顶部导航区、菜单路径导航和内容展示区
- 顶部导航和菜单路径导航与系统首页布局保持一致
- 内容区域根据栏目布局自行配置添加纵向滚动条

##### 3.2.2 普通列表展示

- 普通列表页面内容展示区占页面100%，左右留白间距为16px
- 列表使用"隔行变色"的交互展示方式，便于用户对信息进行有效区分
- 在表格中设置显示最多字数限制。文本显示未完全时，末尾文字增加"…"符号
- 鼠标悬浮时提示完整文本

##### 3.2.3 树形列表

- 树形列表页面内容区域可分为左右结构，左侧树形导航区应按照栏目布局占格1列；右侧列表区占格7列
- 树形导航应突出其层级关系，使用图标来展开和收起下级节点
- 树形导航超过三级时，应提供导航信息搜索，帮助用户快速定位导航信息
- 右侧列表展示信息应便于用户扫视，应根据业务需求进行排序，在必要的情况下相关的内容应连续显示，使信息分类有效以及便于用户识别
- 分页导航应在列表下方显示，位置居中，每页应显示的条数、总条数

##### 3.2.4 弹框

- 弹窗页面分为大、小两种尺寸：
  - 大尺寸：占页面的75%，左右留白间距24px
  - 小尺寸：占页面的30%

**大尺寸弹框**：

- 应用于信息展示类操作、信息填报类操作
- 复杂信息分组独立展示，信息层次符合业务逻辑，方便用户阅读和理解
- 必填项有标识符号，且统一放置于标签左侧，默认使用红色"*"符号表示
- 表单中"流程结束性"按钮放置于表单下方，如"提交"、"保存"按钮
- 填报信息较多，可出现纵向滚动条

**小尺寸弹框**：

- 应用于提示确认类操作
- 当提交成功、重要信息删除、误操作、中断操作等情况时，必须提供提示信息
- 信息提示中有使用信息提示图标，且置于文字内容前
- 信息提示内容过多时，应折行显示

##### 3.2.5 缺省页面

- 当前页面响应异常、没有数据时，出现提示信息，避免用户造成不安的情绪
- 缺省页面设置引导用户下一步操作按钮

##### 3.2.6 加载状态

- 当前页面刷新、界面加载，给予用户加载提示，避免用户长时间等待
- 加载提示包括动态图标、进度条和提示文字

#### 3.3 全屏切换功能规范

- **全屏入口**：在顶部导航区右侧添加全屏切换按钮，图标清晰可见
- **全屏状态**：进入全屏模式后，自动隐藏浏览器UI元素，最大化内容展示区域
- **退出方式**：提供多种退出全屏的方式：ESC键、点击退出按钮、F11键
- **全屏适配**：全屏模式下确保所有页面元素自适应调整，保持良好的视觉效果和交互体验
- **快捷键支持**：支持F11键快速切换全屏模式，并在首次进入时显示快捷键提示
- **状态记忆**：记住用户的全屏偏好设置，在页面刷新后保持之前的选择
- **响应式调整**：全屏模式下，图表和数据可视化组件自动调整尺寸以充分利用屏幕空间
- **交互提示**：进入全屏模式后，首次使用时显示简短的操作提示，3秒后自动消失
