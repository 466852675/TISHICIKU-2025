## 国网PC端页面设计规范

### 1. 总体设计原则

#### 1.1 四大基本原则

（1）**一致性原则**：公司各子信息系统界面应保持色彩、字体、布局、间距等风格统一，融合为视觉一致、交互一致的公司统一信息系统。

（2）**合理性原则**：信息系统布局应遵循用户从上而下，自左向右浏览的操作习惯，页面模块化划分，常用业务功能模块邻近排列。页面设计应横平竖直、结构清晰、层次分明，页面元素之间符合间距要求、不应过于拥挤。

（3）**简洁性原则**：信息系统界面应避免过度装饰和冗余设计，保持界面简洁明了。页面展示必要的信息和功能，减少用户的认知负荷。

（4）**灵活性原则**：信息系统界面应能够适应不同的用户需求和使用环境，满足不同用户个性化需求，实现千人千面的用户体验。

#### 1.2 设计主题

业务管理类系统只有PC端，应采用国网绿和科技蓝2套界面设计，支持主题切换功能。

### 2. 技术规范

#### 2.1 屏幕适配

- PC端系统界面分辨率应适配显示屏比例 16:9
- 适配尺寸应兼容 1920x1080、3840x2160像素及以上显示屏

#### 2.2 色彩规范

采用国网绿、科技蓝主题样式，支持主题切换功能。色彩系统包括四大类：主色、辅助色、功能色和中性色。

##### 2.2.1 主色规范

主色应用于图标、主要按钮，顶部导航背景、超链接文字、高亮提醒等。

| 颜色名称 | 色值              | 应用场景                     |
| -------- | ----------------- | ---------------------------- |
| 国网绿   | #007D3B / #009380 | 主要用于国网绿主题的核心元素 |
| 科技蓝   | #1D7AD9           | 主要用于科技蓝主题的核心元素 |

##### 2.2.2 辅助色规范

辅助色应用于次要按钮、标签、图表、提示信息等。

| 颜色名称 | 色值              | 应用场景                         |
| -------- | ----------------- | -------------------------------- |
| 橙色     | #F77E22 / #FD7C3F | 用于提醒、警告相关的辅助元素     |
| 蓝色     | #4F9DFA / #4AAEFF | 用于信息、提示相关的辅助元素     |
| 青色     | #A6E2FA / #00C1FA | 用于特殊状态或高亮显示的辅助元素 |
| 紫色     | #A67F9E / #AC79F8 | 用于分类、标识的辅助元素         |

##### 2.2.3 功能色规范

功能色应用于向用户明确的传达成功、警告、错误等信息和状态。

| 颜色名称 | 色值              | 应用场景                     |
| -------- | ----------------- | ---------------------------- |
| 成功     | #1DAD73 / #56BE85 | 用于表示操作成功的状态       |
| 提醒     | #FF754C / #FDC748 | 用于表示需要注意的提醒状态   |
| 错误     | #F23A3A / #F8624D | 用于表示操作错误或异常状态   |
| 禁止     | #E53935 / #B3BEC8 | 用于表示禁止、不可操作的状态 |

##### 2.2.4 中性色规范

中性色应用于文字、背景、边框、分割线等。

| 颜色名称 | 色值              | 应用场景                         |
| -------- | ----------------- | -------------------------------- |
| 深色文字 | #333333 / #071717 | 用于标题、重要文本内容           |
| 灰色文字 | #666666 / #505660 | 用于正文、普通文本内容           |
| 浅灰文字 | #999999 / #B0B8C0 | 用于辅助说明、提示文本           |
| 更深背景 | #F2F2F2 / #D9E3E0 | 用于区块背景、分割区域           |
| 浅灰背景 | #F8F8F8           | 用于页面背景、容器背景           |
| 白色背景 | #FFFFFF           | 用于卡片、弹窗等元素背景         |

| 占位文字 | #C8CDD4           | 用于未激活状态的占位提示文字     |
| 反色正文 | #FFFFFF           | 用于深色背景或控制按钮的反色文字 |

##### 2.2.5 色彩应用原则

1. **一致性原则**：同一系统中，相同功能的元素应使用相同的颜色
2. **层级分明**：通过颜色的深浅、明暗来区分信息的层级和重要性
3. **对比度合理**：确保文本与背景之间的对比度符合可读性要求
4. **主题切换**：支持国网绿和科技蓝两套主题的无缝切换

#### 2.3 字体规范

- **中文字体**：微软雅黑、时尚简黑体
- **英文字体**：Arial
- **数字字体**：DIN / DINPro
- **特殊字体**：以图片形式输出
- **行高设置**：字号的1.3-1.5倍
- **字重规范**：常规（Regular）、粗体（Bold）

**字号规范**：

- 系统名称：24px，粗体（Bold）
- 一级菜单：20px，粗体（Bold）
- 模块标题：16px，粗体（Bold）
- 表格内容、输入框文字等正文内容：14px，常规（Regular）
- 辅助文字、占位文字：14px，常规（Regular）

#### 2.4 图标按钮规范

- 按钮风格保持一致
- 主要操作采用面性风格
- 次要操作采用线性风格
- 圆角弧度为5px

### 3. 看板特定规范

#### 3.1 页面框架设计

- **整体结构**：上下结构布局
- **顶部导航**：固定区域，包含Logo、系统名称、日期时间、退出按钮
- **主视觉区域**：核心监控内容展示
- **指标数据区域**：两侧指标展示，与页面间距24px
- **栏目布局**：8列×4排网格系统，栏目间距16px

#### 3.2 国网标准数据可视化组件

- **指标卡片**：关键指标的数值展示和趋势图表
- **实时图表**：折线图、柱状图、饼图等标准图表组件
- **地图组件**：地理信息展示和区域数据分布
- **仪表盘**：圆形、半圆形等仪表盘样式
- **数据表格**：隔行变色，支持排序和筛选

#### 3.3 实时监控仪表盘布局

- **系统首页**：一屏展示所有内容，无滚动条
- **模块导航**：两侧数量对称展示
- **指标看板**：顶部导航 + 指标展示区
- **悬浮菜单**：多层级菜单，包含默认和悬浮状态
- **全屏适配**：浏览器全屏下完整显示

#### 3.4 全屏切换功能规范

- **全屏入口**：顶部导航栏右侧提供全屏切换按钮，图标清晰易识别
- **全屏状态**：隐藏浏览器地址栏、工具栏等界面元素，最大化展示内容区域
- **退出全屏**：支持ESC键退出全屏，同时在页面右下角提供退出全屏按钮
- **全屏适配**：全屏模式下自动调整布局比例，确保各组件合理分布
- **快捷键支持**：支持F11键快速进入/退出全屏模式
- **状态记忆**：记住用户的全屏偏好设置，页面刷新后保持用户选择
- **响应式调整**：全屏模式下图表、数据表格等组件自动调整尺寸以适应更大显示区域
- **交互提示**：首次进入全屏时提供简短的操作提示，增强用户体验

#### 3.5 预警信息展示规范

- **预警等级**：按照功能色规范区分不同等级
- **告警指标**：突出显示，使用对比色彩
- **状态标识**：成功、警告、错误的视觉区分
- **实时更新**：动态数据刷新和状态变化提示

#### 3.6 数据分析图表交互设计

- **图表联动**：多图表间的数据关联和交互
- **钻取功能**：从概览到详细数据的层级展示
- **时间轴控制**：历史数据回放和时间段选择
- **数据筛选**：多维度数据过滤和查询
- **导出功能**：图表和数据的导出能力

#### 3.7 符合规范的色彩编码体系

- **数据状态色彩**：正常、异常、告警的标准色彩
- **图表配色方案**：符合国网视觉规范的图表色彩
- **交互状态色彩**：悬浮、选中、禁用等状态的色彩反馈
