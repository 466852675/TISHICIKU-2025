## 💻 各IDE详细配置指南

### Trae IDE（推荐 ⭐⭐⭐⭐⭐）

**优势**：中文友好、功能完整、67个Skill全支持

**配置方式**：
1. **个人规则**：`Settings` → `Rules` → 添加 [04-规则配置/通用配置/个人规则.md](04-规则配置/通用配置/个人规则.md)
2. **项目规则**：`.trae/rules` 目录下添加项目规则
3. **智能体**：点击AI对话框"创建智能体"，粘贴 [06-提示词库](06-提示词库/) 专家提示词
4. **MCP服务**：`Settings` → `MCP` → 添加 [05-服务配置/MCP服务/通用mcp配置.json](05-服务配置/MCP服务/通用mcp配置.json)

### Cursor IDE（⭐⭐⭐⭐）

**优势**：Composer强大、生态完善

**配置方式**：
1. **全局规则**：`Settings` → `General` → `AI Rules` → 添加个人规则
2. **项目规则**：项目根目录创建 `.cursorrules` 文件
3. **专家提示词**：对话开始时发送或使用 `.cursorrules`
4. **MCP服务**：`Settings` → `Cursor Settings` → `MCP` → 添加配置

### Windsurf IDE（⭐⭐⭐）

**优势**：免费、VS Code生态

**配置方式**：
1. **Cascade配置**：在Cascade面板中配置系统提示词
2. **项目规则**：使用 `.windsurfrules` 文件
3. **专家提示词**：对话中直接发送

### GitHub Copilot（⭐⭐⭐）

**优势**：GitHub生态、IDE集成

**配置方式**：
1. **VS Code设置**：`Settings` → 搜索 `github.copilot`
2. **内联聊天**：使用 `Ctrl+I` 打开内联聊天
3. **专家提示词**：在聊天中发送或使用代码注释

## 🔧 常见问题

### Q1: 本套件支持哪些IDE？

**A**: 主要支持 Trae、Cursor、Windsurf、GitHub Copilot。其中：
- **Trae**: 完整支持所有功能（68个Skill全支持）⭐推荐
- **Cursor**: 支持大部分功能（~40个Skill）
- **Windsurf**: 支持基础功能（~30个Skill）
- **Copilot**: 支持核心功能（~25个Skill）

### Q2: 如何选择合适的专家？

**A**: 参考以下决策树：
```
任务类型？
├── 需求分析 → 需求生成专家 / 可研生成专家 / PRD生成专家
├── 原型设计 → 
│   ├── PC端 → 看板风格 / 系统风格 / 简约风格
│   └── APP端 → 国网风 / 弥散风 / 苹果风
├── 测试验证 → 功能测试 / 自动化测试 / 文档质量校验
├── PPT制作 → 06-PPT设计/ 选择对应风格专家
└── 通用任务 → PPT生成 / 周报生成 / 提示词生成 / 网页生成
```

### Q3: Skill和专家有什么区别？

**A**: 
- **专家（27个）**: 定义AI的角色和行为模式，通过提示词实现
- **Skill（68个）**: 提供具体功能能力，通过MCP或脚本实现
- **关系**: 专家可以调用Skill来完成任务

### Q4: 如何更新到最新版本？

**A**: 
1. 备份现有配置
2. 下载最新版本
3. 对比更新日志，合并自定义配置
4. 测试关键功能

### Q5: 遇到配置问题怎么办？

**A**: 
1. 查阅 [01-完整指南/第7章](01-完整指南/) 常见问题
2. 查看对应模块的 CLAUDE.md
3. 参考 [08-实战案例](08-实战案例/) 中的示例
