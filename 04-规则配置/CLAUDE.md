# 04-规则配置

## 模块定位

本模块定义AI执行过程中的行为约束与工作流程规则，用于保证输出质量、一致性与可交付性。支持Trae、Cursor、Windsurf、GitHub Copilot等主流AI编程IDE。

## 规则体系

```
04-规则配置/
├── 通用配置/          # 全局规则（所有项目生效）
│   └── 个人规则.md    # 个人工作习惯与行为约束
└── 项目规则/          # 项目级规则（当前项目生效）
    └── 6A执行规则.md  # 6A工作流规范
```

## 规则优先级

当多条规则同时存在时，按以下优先级执行：

1. **对话中的明确指令**（最高优先级）
2. **项目规则**（6A执行规则）
3. **个人规则**（全局行为约束）
4. **IDE默认行为**（最低优先级）

## 快速配置

### Trae用户（推荐）

**步骤1：配置个人规则**
1. 打开 Trae IDE
2. AI对话框 → 设置 ⚙️ → 规则 → 个人规则
3. 创建 `user_rules.md`
4. 复制 `通用配置/个人规则.md` 中的"通用规则内容"
5. 保存

**步骤2：配置项目规则**
1. 打开项目
2. AI对话框 → 设置 ⚙️ → 规则 → 项目规则
3. 创建 `project_rules.md`
4. 复制 `项目规则/6A执行规则.md` 中的"6A工作流规则内容"
5. 保存

### Cursor用户

**步骤1：配置个人规则**
1. 打开 Cursor Settings → General → AI Rules
2. 添加全局规则内容

**或在项目根目录创建 `.cursorrules` 文件**：
```markdown
# 合并个人规则 + 6A工作流规则
[粘贴个人规则内容]

[粘贴6A工作流规则内容]
```

### 其他IDE用户

1. 将规则内容保存为项目文档
2. 在对话开始时引用规则文档
3. 或在每次对话开头粘贴规则内容

## 规则验证

配置完成后，验证规则是否生效：

```
请介绍一下你自己，并说明你遵循什么规则
```

如果AI回复中提到了配置的内容（如"使用中文"、"6A工作流"等），说明规则配置成功。

## 使用场景

### 场景1：个人开发

- 配置个人规则，统一所有项目的AI行为
- 根据项目特点选择性配置6A工作流

### 场景2：团队协作

- 统一团队个人规则，确保代码风格一致
- 统一项目规则，确保工作流程一致
- 将规则文件提交到代码库，新成员自动继承

### 场景3：多IDE协作

- 将规则内容同步到各IDE配置
- Trae用户：智能体 + 规则配置
- Cursor用户：.cursorrules 文件
- 其他IDE：项目文档引用

## 最佳实践

### 1. 规则分层原则

- **个人规则**：放通用的、跨项目的行为约束
- **项目规则**：放项目特定的、6A工作流相关的内容
- **对话指令**：放当前任务的特定要求

### 2. 规则迭代优化

- 根据实际使用情况调整规则
- 收集团队反馈，持续优化
- 定期回顾规则的有效性

### 3. 规则文档化

- 规则变更要记录原因
- 重要规则添加使用示例
- 复杂规则添加说明注释
