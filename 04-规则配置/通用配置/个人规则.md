# AI编程IDE个人规则配置

> 🎯 **全局AI行为约束** - 适用于Trae、Cursor、Windsurf、GitHub Copilot等主流AI编程IDE

---

## 首要原则

- 始终使用中文回答
- 所有工件 artifacts（包括计划 plans、任务 tasks、验证报告 walkthroughs 和源代码提交信息 commit messages）都必须使用中文编写
- 复杂任务先拆分再执行
- 完成任务后明确说明"写好了"或"已完成"

---

## 各IDE配置方式

### Trae IDE

**配置路径**：AI对话框 → 右上角设置 ⚙️ → 规则 → 个人规则 → 创建 user_rules.md

**操作步骤**：
1. 点击"+ 创建 user_rules.md"按钮
2. 复制下方"通用规则内容"到编辑器
3. 按 `Ctrl+S` 保存
4. 规则立即生效，无需重启

### Cursor

**配置路径**：Settings → General → AI Rules

**或在项目根目录创建 `.cursorrules` 文件**：
```markdown
# Cursor 个人规则
- 所有回复使用中文
- 代码添加中文注释
- 优先使用现代JavaScript/TypeScript语法
- 复杂任务先拆分再执行
```

### 其他IDE

在IDE设置中找到AI/智能助手配置，添加以下规则内容。

---

## 通用规则内容

```markdown
# 个人规则

## 语言与沟通
- 所有回复使用中文
- 技术术语可保留英文（如API、JSON、React）
- 解释复杂概念时使用通俗易懂的语言
- 主动询问不确定的需求

## 代码规范
- 所有代码注释使用中文
- 变量名使用英文，语义清晰
- 函数名使用驼峰命名法（camelCase）
- 类名使用大驼峰命名法（PascalCase）
- 常量使用全大写+下划线（UPPER_SNAKE_CASE）
- 文件名使用小写+下划线（lowercase_snake_case）
- 注释密度保持在30%以上

## 开发习惯
- 复杂任务先拆分步骤，逐步执行
- 每完成一个步骤，简要说明完成情况
- 遇到错误时，先分析原因再提供解决方案
- 提供多种解决方案时，说明各自的优缺点
- 完成任务后明确说"写好了"或"已完成"

## 代码质量
- 优先使用现代语法和最佳实践
- 代码简洁，避免冗余
- 考虑边界情况和异常处理
- 提供可运行的完整代码示例
- 代码块中标注关键步骤的注释

## 安全与隐私
- 不生成包含敏感信息的代码
- 提醒用户修改默认密码和密钥
- 建议使用环境变量管理配置
- 避免在代码中硬编码敏感数据

## 学习与改进
- 解释为什么选择某种实现方式
- 提供相关文档链接供深入学习
- 鼓励用户提出改进建议
- 根据反馈持续优化输出质量
```

## 规则验证

配置完成后，在AI对话框中输入：

```
请介绍一下你自己，并说明你遵循什么规则
```

如果AI回复中提到了"使用中文"、"添加注释"等内容，说明规则配置成功。

## 规则更新

当需要更新规则时：

1. **Trae**：直接编辑 user_rules.md 文件，保存后立即生效
2. **Cursor**：编辑 Settings 中的 AI Rules 或 `.cursorrules` 文件
3. **其他IDE**：在对应设置中修改

## 常见问题

### Q1: 规则配置后不生效？

**解决方案**：
- 检查文件是否保存（Ctrl+S）
- 重新打开AI对话框
- 确认文件路径正确
- 重启IDE后重试

### Q2: 如何针对不同项目使用不同规则？

**解决方案**：
- **Trae**：使用项目规则（project_rules.md）覆盖个人规则
- **Cursor**：在项目根目录创建 `.cursorrules` 文件
- **其他IDE**：使用项目级配置

### Q3: 规则冲突时如何处理？

**优先级**（从高到低）：
1. 对话中的明确指令
2. 项目规则
3. 个人规则
4. IDE默认行为
