# 托管代理基础设施

托管代理在远程沙盒环境中运行，而非在本地机器上。设计良好时，它们提供无限并发、一致的执行环境和多人协作。关键洞察是会话速度应仅受模型提供商首token时间限制，所有基础设施设置在用户开始会话之前完成。

## 何时激活

在以下情况下激活此技能：
- 构建独立于用户设备运行的后台编码代理
- 为代理工作负载设计沙盒执行环境
- 实现具有共享状态的多人代理会话
- 创建多客户端代理接口（Slack、Web、Chrome扩展）
- 扩展超出本地机器约束的代理基础设施
- 构建代理生成子代理以进行并行工作的系统

## 核心概念

托管代理解决本地代理执行的基本限制：资源争用、环境不一致和单用户约束。通过将代理执行移动到远程沙盒环境，团队获得无限并发、可重现的环境和协作工作流。

架构由三层组成：用于隔离执行的沙盒基础设施、用于状态管理和客户端协调的API层，以及用于跨平台用户交互的客户端接口。每层都有特定的设计要求，使系统能够扩展。

## 详细主题

### 沙盒基础设施

**核心挑战**
快速启动完整的开发环境是主要的技术挑战。用户期望近乎即时的会话启动，但开发环境需要克隆仓库、安装依赖和运行构建步骤。

**镜像注册表模式**
定期预构建环境镜像（每30分钟运行良好）。每个镜像包含：
- 已知提交处的克隆仓库
- 所有运行时依赖已安装
- 初始设置和构建命令已完成
- 从运行应用程序和测试套件一次缓存的文件

启动会话时，从最新镜像启动沙盒。仓库最多落后30分钟，使与最新代码的同步更快。

**快照和恢复**
在关键点获取文件系统快照：
- 初始镜像构建后（基础快照）
- 代理完成更改时（会话快照）
- 沙盒退出前以进行潜在跟进

这使跟进提示能够即时恢复，而无需重新运行设置。
