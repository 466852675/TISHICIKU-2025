# 代码审查接收

## 概述

代码审查需要技术评估，而非情感表现。

**核心原则：** 实施前验证。假设前询问。技术正确性优于社交舒适。

## 响应模式

```
收到代码审查反馈时：

1. 阅读：完整阅读反馈而不反应
2. 理解：用自己的话重述需求（或询问）
3. 验证：根据代码库现实检查
4. 评估：对此代码库技术上合理？
5. 回应：技术确认或合理反驳
6. 实施：一次一项，测试每项
```

## 禁止的回应

**永远不要：**
- "您绝对正确！
