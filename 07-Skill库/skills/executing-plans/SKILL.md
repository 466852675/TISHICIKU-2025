# 执行计划

## 概述

加载计划，批判性审查，分批执行任务，在批次之间报告以进行审查。

**核心原则：** 带有架构师审查检查点的分批执行。

**开始时宣布：** "我正在使用executing-plans技能来实施此计划。"

## 过程

### 第1步：加载和审查计划
1. 阅读计划文件
2. 批判性审查 - 确定关于计划的任何问题或疑虑
3. 如果有疑虑：在开始之前与您的合作伙伴提出
4. 如果没有疑虑：创建TodoWrite并继续

### 第2步：执行批次
**默认：前3个任务**

对于每个任务：
1. 标记为in_progress
2. 完全遵循每个步骤（计划有小步骤）
3. 按指定运行验证
4. 标记为completed

### 第3步：报告
当批次完成时：
- 显示实施的内容
- 显示验证输出
- 说："准备好反馈。"

### 第4步：继续
基于反馈：
- 如果需要应用更改
- 执行下一批次
- 重复直到完成

### 第5步：完成开发

所有任务完成并验证后：
- 宣布："我正在使用finishing-a-development-branch技能来完成此工作。"
- **必需子技能：** 使用superpowers:finishing-a-development-branch
- 遵循该技能验证测试、呈现选项、执行选择

## 何时停止并寻求帮助

**立即停止执行当：**
- 批次中遇到障碍（缺少依赖、测试失败、指令不清楚）
- 计划有严重差距阻止开始
- 您不理解指令
- 验证反复失败

**询问澄清而非猜测。**

## 何时重新审视早期步骤

**当以下情况时返回审查（第1步）：**
- 合作伙伴根据您的反馈更新计划
- 基本方法需要重新思考

**不要强行通过障碍** - 停止并询问。

## 记住
- 首先批判性审查计划
- 完全遵循计划步骤
- 不要跳过验证
- 计划说要使用时参考技能
- 批次之间：只报告并等待
- 受阻时停止，不要猜测
