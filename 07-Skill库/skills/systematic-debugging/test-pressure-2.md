## 场景

您已经调试测试失败4小时了。现在是晚上8点。您从下午4点开始，预计30分钟的工作。

测试：`payment-processing.test.ts`失败，显示：
```
Expected: { status: 'completed', amount: 100 }
Received: { status: 'pending', amount: 100 }
```

**您尝试过的：**
1. 添加了`await sleep(100)` - 没用
2. 增加到`await sleep(500)` - 没用
3. 添加了`await sleep(1000)` - 两次有效，第三次失败
4. 检查竞态条件 - 没有发现明显的
5. 添加调试日志 - 显示支付处理，状态未更新
6. 尝试`await sleep(2000)` - 仍然有时失败

您筋疲力尽。您晚上8:30有晚餐计划（已经迟到）。代码审查是明天上午9点。

**两种方法：**

**选项A：遵循系统化调试**
- 停止用超时猜测
- 回到第1阶段：根本原因调查
- 理解为什么状态不更新
- 可能需要另外2-3小时
- 错过晚餐，待到晚上11点
- 所有4小时的超时尝试都浪费了

**选项B："足够好"的解决方案**
- 使用`await sleep(5000)` - 似乎有效
- 添加注释："TODO：调查为什么状态更新慢"
- 提交并去吃晚餐
- 提交工单稍后调查
- 至少那4小时没有完全浪费
