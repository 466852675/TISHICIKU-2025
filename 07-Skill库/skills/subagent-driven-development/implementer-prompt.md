# 实施者子代理提示模板

在调度实施者子代理时使用此模板。

```
Task tool (general-purpose):
  description: "实施任务N：[任务名称]"
  prompt: |
    您正在实施任务N：[任务名称]

    ## 任务描述

    [计划中任务的完整文本 - 粘贴在此处，不要让子代理读取文件]

    ## 上下文

    [场景设置：这适合哪里，依赖关系，架构上下文]

    ## 开始之前

    如果您对以下方面有问题：
    - 要求或验收标准
    - 方法或实施策略
    - 依赖关系或假设
    - 任务描述中任何不清楚的内容

    **现在就问。** 在开始工作之前提出任何疑虑。

    ## 您的工作

    一旦您清楚了要求：
    1. 准确实施任务指定的内容
    2. 编写测试（如果任务要求则遵循TDD）
    3. 验证实施有效
    4. 提交您的工作
    5. 自我审查（见下文）
    6. 报告回来

    工作目录：[目录]

    **工作时：** 如果您遇到意外或不清楚的内容，**提问**。
    暂停并澄清总是可以的。不要猜测或做假设。

    ## 报告之前：自我审查

    用新的眼光审查您的工作。问自己：

    **完整性：**
    - 我是否完全实施了规范中的所有内容？
    - 我是否遗漏了任何要求？
    - 是否有我未处理的边缘情况？

    **质量：**
    - 这是我最好的工作吗？
    - 名称是否清晰准确（匹配功能，而非实现方式）？
    - 代码是否干净且可维护？

    **纪律：**
    - 我是否避免了过度构建（YAGNI）？
    - 我是否只构建了请求的内容？
    - 我是否遵循了代码库中的现有模式？

    **测试：**
    - 测试是否实际验证行为（而非仅模拟行为）？
    - 如果要求，我是否遵循了TDD？
    - 测试是否全面？

    如果在自我审查期间发现问题，请在报告之前立即修复。

    ## 报告格式

    完成后，报告：
    - 您实施的内容
    - 您测试的内容和测试结果
    - 更改的文件
    - 自我审查发现（如有）
    - 任何问题或疑虑
```
