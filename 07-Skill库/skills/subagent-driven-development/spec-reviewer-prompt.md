# 规范合规审查提示模板

在调度规范合规审查子代理时使用此模板。

**目的：** 验证实施者构建了所请求的内容（不多不少）

```
Task tool (general-purpose):
  description: "审查任务N的规范合规性"
  prompt: |
    您正在审查实施是否符合其规范。

    ## 请求的内容

    [任务要求的完整文本]

    ## 实施者声称他们构建的内容

    [来自实施者的报告]

    ## 关键：不要相信报告

    实施者完成得太快，令人怀疑。他们的报告可能不完整、
    不准确或过于乐观。您必须独立验证一切。

    **不要：**
    - 相信他们关于实施内容的说法
    - 相信他们关于完整性的声明
    - 接受他们对要求的解释

    **要：**
    - 阅读他们实际编写的代码
    - 将实际实施与要求逐行比较
    - 检查他们声称实施但缺失的部分
    - 查找他们未提及的额外功能

    ## 您的工作

    阅读实施代码并验证：

    **缺失的要求：**
    - 他们是否实施了所有请求的内容？
    - 是否有他们跳过或遗漏的要求？
    - 他们是否声称某物有效但实际上未实施？

    **额外/不需要的工作：**
    - 他们是否构建了未请求的内容？
    - 他们是否过度工程化或添加了不必要的功能？
    - 他们是否添加了规范中未包含的"锦上添花"功能？

    **误解：**
    - 他们是否以不同于意图的方式解释要求？
    - 他们是否解决了错误的问题？
    - 他们是否实施了正确的功能但方式错误？

    **通过阅读代码验证，而非相信报告。**

    报告：
    - ✅ 符合规范（如果在代码检查后一切匹配）
    - ❌ 发现的问题：[具体列出缺失或额外的内容，带有文件:行号引用]
```
