# 06-提示词库

## 模块定位

本模块是专家提示词资产库，覆盖需求分析、原型设计、测试验证、通用文档与图表/PPT风格等能力。支持Trae、Cursor、Windsurf、GitHub Copilot等主流AI编程IDE。

## 什么是提示词专家？

提示词专家是封装好的AI角色配置，定义了AI在特定领域的专业行为、知识范围和输出规范。就像为AI聘请了一位专业顾问。

**提示词专家可以做什么？**
- 📋 **需求分析**：将模糊需求转化为标准文档
- 🎨 **原型设计**：生成符合规范的界面代码
- 🧪 **测试验证**：检查质量和发现问题
- 📊 **文档制作**：生成PPT、周报等办公文档
- 📈 **图表设计**：绘制流程图、架构图

## 各IDE使用方式

### Trae IDE（推荐）

**方式1：创建专用智能体（推荐）**

1. 在Trae中点击"创建智能体"
2. 命名智能体（如"国网APP设计专家"）
3. 从本模块复制对应专家的提示词内容
4. 粘贴到智能体配置中
5. 保存并开始使用

**方式2：项目规则配置**

1. 在项目根目录创建 `.trae/rules/project_rules.md`
2. 将专家提示词作为项目规则的一部分
3. 在AI对话框中直接调用

### Cursor

**方式1：项目级专家（推荐）**

1. 在项目根目录创建 `.cursorrules` 文件
2. 复制专家提示词内容到文件
3. Cursor会自动应用该专家角色

**方式2：对话级专家**

1. 在AI聊天中先发送系统Prompt：
```
你现在是"国网APP设计专家"，请按照以下规则回复：
[粘贴专家提示词内容]
```

**方式3：全局配置**

1. 打开 Cursor Settings → General → AI Rules
2. 将常用专家提示词添加到全局规则

### 其他IDE

**方式1：对话开始时发送**

在每次AI对话开始时，先发送专家提示词：
```
你现在是[专家名称]，具备以下专业能力：
[粘贴专家提示词内容]
```

**方式2：项目文档引用**

将专家提示词保存为项目文档，在对话中引用：
```
请参考 @需求生成专家.md 的规范进行需求分析
```

## 典型工作流程

### 企业管理系统开发流程

```
需求A单 → 需求生成专家 → PRD生成专家 → 系统风格专家 → 功能测试专家 → 自动化测试专家 → 文档质量校验专家
```

**各IDE专家切换**：
- **Trae**：在智能体下拉菜单中切换
- **Cursor**：更换.cursorrules文件或重新发送Prompt
- **其他IDE**：在对话中重新发送专家提示词

### 数据看板开发流程

```
业务需求 → PRD生成专家 → 看板风格专家 → 功能测试专家 → 文档质量校验专家
```

### APP端设计流程

```
APP需求 → 选择风格专家（国网风/弥散风/苹果风）→ UI规范分析 → 功能测试专家 → 文档质量校验专家
```

## 使用技巧

### 1. 专家组合策略

**串行模式**（传统瀑布式）：
需求 → 设计 → 开发 → 测试

**并行模式**（敏捷开发）：
- 需求分析 与 原型设计 并行
- 功能测试 与 文档校验 同步

### 2. 跨IDE协作

当团队成员使用不同IDE时：
1. 统一使用本模块的专家提示词
2. Trae用户创建智能体
3. Cursor用户使用.cursorrules
4. 其他IDE使用对话Prompt
5. 确保输出标准一致

### 3. 提示词优化

- 在专家提示词基础上添加项目特定要求
- 根据实际使用效果调整提示词
- 记录有效的提示词变体
