# 提示词专家库

> 🧠 **智能体配置模板的核心专家库** - 涵盖需求分析、原型设计、测试验证、通用任务、图表设计五大专业领域（共18个专家）
> 
> 本专家库支持：Trae、Cursor、Windsurf、GitHub Copilot 等主流AI编程IDE

---

## 🚀 快速开始（按IDE选择）

### Trae 用户（推荐）

**创建智能体步骤**：
1. 点击AI对话框左下角"创建智能体"
2. 命名智能体（如"国网APP设计专家"）
3. 从下方选择对应专家，复制提示词内容
4. 粘贴到智能体配置中，保存
5. 在对话中选择该智能体即可使用

### Cursor 用户

**配置方式**：
1. **项目级**：在项目根目录创建 `.cursorrules` 文件，复制专家提示词
2. **对话级**：在AI聊天中先发送专家提示词定义角色
3. **全局级**：Settings → AI Rules 中添加常用专家

### 其他IDE用户

**使用方式**：
1. 在AI对话开始时发送专家提示词
2. 或将提示词保存为项目文档，对话中引用

## 💡 使用技巧与最佳实践

### 🎯 专家选择建议

#### 按项目复杂度选择
- **简单项目** (⭐⭐): 简约风格专家、需求生成专家、功能测试专家
- **中等项目** (⭐⭐⭐): 系统风格专家、PRD生成专家、自动化测试专家
- **复杂项目** (⭐⭐⭐⭐+): 看板风格专家、国网风专家、UI规范专家、draw.io图表生成专家

#### 按行业领域选择
- **电力行业**: 国网风专家 + 国网PRD生成专家
- **互联网公司**: 苹果风专家 + 弥散风专家
- **传统企业**: 系统风格专家 + 简约风格专家
- **数据分析**: 看板风格专家 + UI规范专家 + draw.io图表生成专家

#### 按IDE选择
- **Trae**: 创建专用智能体，快速切换
- **Cursor**: .cursorrules项目级配置 + 对话Prompt
- **Windsurf**: 对话Prompt方式
- **GitHub Copilot**: 对话Prompt方式

### 📋 输入要求标准

#### 高质量输入特征
- ✅ **需求明确**: 清晰的功能需求和性能要求
- ✅ **场景完整**: 包含主要使用场景和异常场景
- ✅ **标准格式**: 使用标准的需求文档模板
- ✅ **信息完整**: 包含业务背景、用户群体、技术约束
- ✅ **可量化指标**: 明确的验收标准和性能指标

#### 输入文档准备
1. **业务需求文档**: 详细的业务背景和功能需求
2. **用户画像**: 目标用户群体特征和使用习惯
3. **技术约束**: 技术栈限制和性能要求
4. **设计偏好**: 品牌规范和设计风格要求
5. **验收标准**: 明确的功能验收和质量标准

### 🔗 专家协同工作

#### 串行工作模式
适用于传统瀑布式开发项目：
1. **需求阶段** → 2. **设计阶段** → 3. **开发阶段** → 4. **测试阶段**

#### 并行工作模式
适用于敏捷开发和迭代优化：
- **需求分析** 与 **原型设计** 并行进行
- **功能测试** 与 **文档校验** 同步开展
- **多版本设计** 同时进行方案对比

#### 质量保证机制
- **每个阶段输出**: 明确的交付物和质量标准
- **阶段间检查**: 前一阶段输出作为后一阶段输入
- **反馈循环**: 测试结果反馈到设计和需求阶段

### 🔄 跨IDE协作

当团队成员使用不同IDE时：
1. 统一使用本模块的专家提示词
2. Trae用户创建智能体
3. Cursor用户使用.cursorrules
4. 其他IDE使用对话Prompt
5. 确保输出标准一致
